# 📄 部署方案文档

> **文档说明**：本文档详细描述 Scaffold AI Assistant 的部署方案。核心原则：复用现有脚手架的基础组件（MySQL、Redis、Nacos），仅新增 Milvus 向量数据库。

---

## 9.0 基础组件复用说明

| 组件 | 是否复用 | 说明 |
|-----|---------|------|
| **MySQL** | ✅ 复用 | 使用现有脚手架的 MySQL 数据库，共享 `sys_user` 表（identity字段标识身份，权限通过Gateway的userFrom字段控制） |
| **Redis** | ✅ 复用 | 使用现有脚手架的 Redis，用于 Token 存储、对话记忆、限流、幂等性 |
| **Nacos** | ✅ 复用 | 使用现有脚手架的 Nacos，用于配置管理和服务发现 |
| **Gateway** | ✅ 复用 | 使用现有脚手架的 Gateway，仅需新增路由配置 |
| **Prometheus** | ✅ 复用 | 使用现有脚手架的 Prometheus，用于监控 |
| **Grafana** | ✅ 复用 | 使用现有脚手架的 Grafana，用于可视化 |
| **Milvus** | ❌ 新增 | 向量数据库，现有脚手架未包含，需新增部署 |

## 9.1 端口规范

> **说明**：以下端口来自现有脚手架 `zmbdp-{service}-service-{env}.yaml` Nacos 配置文件，dev 和 prd 环境端口一致。chat-service 新增端口遵循现有命名规律（18081-18085 已被占用，chat-service 使用 18084）。

| 服务 | 端口 | 说明 | 配置来源 |
|-----|------|------|---------|
| **zmbdp-gateway-service** | 10030 | 网关端口（dev/prd 一致） | `zmbdp-gateway-service-{env}.yaml` |
| **zmbdp-admin-service** | 18081 | B端管理服务端口 | `zmbdp-admin-service-{env}.yaml` |
| **zmbdp-portal-service** | 18083 | C端用户服务端口 | `zmbdp-portal-service-{env}.yaml` |
| **zmbdp-file-service** | 18082 | 文件服务端口 | `zmbdp-file-service-{env}.yaml` |
| **zmbdp-mstemplate-service** | 18085 | 模板服务端口 | `zmbdp-mstemplate-service-{env}.yaml` |
| **zmbdp-chat-service** | 18084 | **新增**：AI 服务端口 | `zmbdp-chat-service-{env}.yaml` |
| **MySQL（dev/test 容器内）** | 3306 | 数据库端口 | docker-compose-mid.yml |
| **MySQL（prd 宿主机映射）** | 3308 | 容器内 3306 映射到宿主机 3308 | docker-compose-mid.yml: `"3308:3306"` |
| **Redis（dev/test）** | 6379 | 单节点缓存端口 | docker-compose-mid.yml |
| **Redis（prd）** | 6380-6385 | 缓存集群端口（3主3从，跨2台VM） | `deploy/prd/vm1/app/redis/init.sh` |
| **Milvus** | 19530 | Milvus主端口（gRPC） | docker-compose |
| **Milvus** | 9091 | Milvus管理端口（HTTP） | docker-compose |

## 9.2 Docker部署

> **复用说明**：脚手架现有服务使用 `docker-maven-plugin`（IDEA 的 deploy 功能）构建并部署到测试环境，prd 环境使用 docker-compose 在本地构建。AI 模块遵循相同部署方式。下面给出的是独立部署 chat-service 时可用的配置，**实际部署时建议复用现有脚手架的 deploy 目录结构**（如 `deploy/dev/app/zmbdp-chat-service/`）。

**Dockerfile**（放在 `zmbdp-chat-service/src/main/docker/` 下）：
```dockerfile
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY target/zmbdp-chat-service.jar app.jar
EXPOSE 18084
ENTRYPOINT ["exec", "java", "$JAVA_OPTS", "-jar", "app.jar"]
```

## 9.4 Nacos配置

### 9.4.1 网关路由配置（需在 zmbdp-gateway-service-dev.yaml 中添加）

> **重要**：网关路由路径前缀必须与现有脚手架规则一致：`/portal/**` 路由到 portal-service，`/admin/**` 路由到 admin-service。**不要**使用 `/chat/**` 作为路由前缀，因为 AI 的 C 端接口属于 portal-service，应走 `/portal/chat/**`。

**chat-service 不直接暴露给前端**——所有前端请求经 Gateway 路由到 portal-service 或 admin-service，再通过 Feign/WebClient 内部调用 chat-service。因此**不需要为 chat-service 配置网关路由**。

**需要在 portal-service 和 admin-service 中新增的 Feign 配置**（指向 chat-service）：

```yaml
# 在 portal-service 和 admin-service 的 nacos 配置中添加
spring:
  cloud:
    openfeign:
      client:
        config:
          zmbdp-chat-service:
            url: lb://zmbdp-chat-service
```

**Gateway 路由保持现有配置不变**（参考 `deploy/dev/res/sql/DEFAULT_GROUP/zmbdp-gateway-service-dev.yaml`）：

```yaml
spring:
  cloud:
    gateway:
      routes:
        # 现有路由配置（保持不变）
        - id: zmbdp-portal-service
          uri: lb://zmbdp-portal-service
          predicates:
            - Path=/portal/**
          filters:
            - StripPrefix=1
        - id: zmbdp-admin-service
          uri: lb://zmbdp-admin-service
          predicates:
            - Path=/admin/**
          filters:
            - StripPrefix=1
        # ... 其他现有路由
```

> **路径映射示例**：
> - 前端请求 `http://ip:10030/portal/chat/stream` → Gateway 匹配 `/portal/**` + StripPrefix=1 → portal-service 收到 `/chat/stream`
> - 前端请求 `http://ip:10030/admin/knowledge/sources` → Gateway 匹配 `/admin/**` + StripPrefix=1 → admin-service 收到 `/knowledge/sources`

### 9.4.3 bootstrap.yml配置（应用启动时加载）

> **关键说明**：
> 1. Nacos namespace 使用**字符串名称** `scaffold-ai-assistant-${RUN_ENV}`（如 `scaffold-ai-assistant-dev`），**不是 UUID**
> 2. 共享配置统一命名规则：`share-xxx-${env}.yaml`（**单数 share + 带环境后缀**）
> 3. `share-token-${env}.yaml` 包含 JWT 密钥 `${jwt.token.secret}`，**必须引入**，否则 AuthFilter 启动失败
> 4. Nacos 用户名密码复用脚手架现有配置

```yaml
spring:
  output:
    ansi:
      enabled: ALWAYS
  application:
    name: zmbdp-chat-service
  profiles:
    active: ${RUN_ENV}
  cloud:
    nacos:
      discovery:
        username: nacos
        password: Hf@173503494
        namespace: scaffold-ai-assistant-${RUN_ENV}
        server-addr: ${NACOS_ADDR}
      config:
        username: nacos
        password: Hf@173503494
        namespace: scaffold-ai-assistant-${RUN_ENV}
        server-addr: ${NACOS_ADDR}
        file-extension: yaml
        # 配置共享的配置（与 zmbdp-portal-service / zmbdp-admin-service 保持一致命名规则）
        shared-configs:
          # Redis 配置（必需）
          - data-id: share-redis-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true
          # Caffeine 本地缓存配置（可选，如需要本地缓存）
          - data-id: share-caffeine-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true
          # MySQL 配置（必需）
          - data-id: share-mysql-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true
          # Token 配置（必需，含 jwt.token.secret）
          - data-id: share-token-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true
          # 限流配置（必需，@RateLimit 注解依赖）
          - data-id: share-ratelimit-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true
          # 日志配置（必需）
          - data-id: share-log-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true
          # 监控配置（必需，Prometheus + Grafana）
          - data-id: share-monitor-${spring.profiles.active}.${spring.cloud.nacos.config.file-extension}
            refresh: true

# MyBatis Plus 配置（与现有服务保持一致）
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

> **环境变量说明**：
> - `RUN_ENV`：运行环境（dev/test/prd），与现有服务一致
> - `NACOS_ADDR`：Nacos 地址，开发环境为 `127.0.0.1:8848`，容器环境为 `scaffold-ai-assistant-nacos:8848`

### 9.4.4 开发环境配置（zmbdp-chat-service-dev.yaml）

> **说明**：此文件放在 Nacos 配置中心，命名空间为 `scaffold-ai-assistant-dev`，group 为 `DEFAULT_GROUP`。MySQL/Redis 连接信息实际从 `share-mysql-dev.yaml` 和 `share-redis-dev.yaml` 共享配置中读取，本文件只配置 chat-service 独有的内容。

```yaml
server:
  port: 18084  # chat-service 专用端口，遵循脚手架 18081-18085 规律

# AI 模型配置（chat-service 独有）
spring:
  ai:
    dashscope:
      api-key: # 你的通义千问API Key（建议通过环境变量 ${DASHSCOPE_API_KEY} 注入，不硬编码）
      chat:
        model: qwen-max
        options:
          temperature: 0.7
          max-tokens: 4096
      embedding:
        model: text-embedding-v1

# Milvus 向量数据库配置（chat-service 独有）
milvus:
  host: 127.0.0.1
  port: 19530

# 知识库路径配置（chat-service 独有，供 ReadFileTool 等工具使用）
knowledge:
  base-path: /path/to/scaffold-ai-assistant
  # 路径白名单支持动态调整，范围包含整个项目（参考 05-Agent工具设计.md）
  allowed-paths:
    - /path/to/scaffold-ai-assistant
```

## 9.6 健康检查配置

### 9.6.1 Actuator端点

| 端点 | 路径 | 说明 |
|-----|------|------|
| **健康检查** | `/actuator/health` | 服务健康状态 |
| **就绪检查** | `/actuator/health/readiness` | 服务就绪状态 |
| **存活检查** | `/actuator/health/liveness` | 服务存活状态 |
| **指标** | `/actuator/metrics` | 应用指标 |

```yaml
management:
  endpoints:

        include: health, metrics, info
  endpoint:
    health:
      show-details: always
      probes:
        enabled: true
```

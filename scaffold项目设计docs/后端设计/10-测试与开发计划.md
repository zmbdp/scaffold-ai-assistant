## 10.2 开发计划

### 10.2.1 阶段划分

| 阶段 | 时间 | 目标 | 交付物 |
|-----|------|------|--------|
| **Phase 1** | 第1-2周 | 基础架构搭建 | Spring Boot项目、依赖配置、Milvus/Redis/MySQL集成 |
| **Phase 2** | 第3-4周 | RAG能力实现 | IKnowledgeLoaderService、IVectorStoreService、知识同步 |
| **Phase 3** | 第5-6周 | Agent工具实现 | 7个Agent工具全部实现、ToolRegistryService注册 |

| **Phase 5** | 第9-10周 | B端管理后台 | KnowledgeController等、IAdminService、扩展AuthFilter、工具管理、召回测试、系统健康检测 |
| **Phase 6** | 第11-12周 | 测试与优化 | 黄金测试问题集验证、性能优化、Bug修复 |

### 10.2.3 每周任务分解

#### Phase 1：基础架构搭建（第1-2周）

**第1周**：
- 在现有脚手架项目基础上新增 `zmbdp-chat` 模块（父 pom + zmbdp-chat-api + zmbdp-chat-service）
- 添加Maven依赖（Spring AI Alibaba 1.0.0 GA（1.0.0.2）、Milvus SDK、JavaParser）
- 复用脚手架共享配置（share-redis、share-mysql、share-token、share-ratelimit 等）
- 配置 bootstrap.yml（Nacos namespace、shared-configs）
- 搭建本地开发环境（Milvus Docker 启动，Redis/MySQL 复用现有）

**第2周**：
- 实现数据库初始化脚本（schema.sql，AI模块新增表）
- 实现实体类和Mapper层（复用 MybatisPlusConfig）
- 复用脚手架全局异常处理（GlobalExceptionHandler）
- 验证所有组件连接正常

---

#### Phase 2：RAG能力实现（第3-4周）

**第3周**：
- 实现 IKnowledgeLoaderService（文档加载、分块处理，含分布式锁防并发）
- 实现文档分块策略（Markdown、JavaDoc、配置文件、Java源码，JavaParser按类/方法分块）
- 实现 Embedding 模型集成

**第4周**：
- 实现 IVectorStoreService（向量存储、检索，Milvus 调用）
- 实现 Milvus 集合创建和索引配置
- 实现知识同步功能（MySQL标记DELETED + Milvus物理删除的软删除机制）
- 验证RAG问答效果

---

#### Phase 3：Agent工具实现（第5-6周）

**第5周**：
- 实现ReadFileTool、SearchCodeTool、ListDirTool
- 实现SearchInFileTool、NacosConfigTool
- 实现工具注册和配置

**第6周**：
- 实现CompareConfigTool、PreDeployCheckTool
- 实现工具安全性校验（路径白名单）
- 实现工具调用流程集成
- 验证所有工具功能

---

#### Phase 4：C端对话服务（第7-8周）

**第7周**（chat-service 侧）：
- 实现 IChatMemoryService（对话记忆，Redis 存储，addMessage/getHistory）
- 实现 IChatService（核心对话逻辑，streamChat 8步执行流程）
- 实现 RAG 检索与工具调用的协调（Feign 调用 retrieveContext + ToolRegistryService 工具列表）

**第8周**（portal-service 侧 + 反馈机制）：
- 实现 PortalChatService（C端流式对话处理，WebClient 配置、8步执行流程、RAG 检索与 Prompt 拼接）
- 实现 PortalChatController（流式对话接口，`/chat/completions/stream`、`/chat/image/completions/stream`，WebClient + SSE）
- 实现 HistoryController（历史记录接口，Feign 调用 HistoryApi，优先 Redis 降级 MySQL）
- 实现回答反馈机制（FeedbackPortalController、IFeedbackService、SysAiFeedback，含点赞/点踩/撤销/查询，详见 03-C端功能设计.md 3.2.5 节）
- 验证C端完整功能

---

#### Phase 5：B端管理后台（第9-10周）

**第9周**：
- 扩展Gateway AuthFilter（支持AI模块B端路径的来源校验，详见 07-项目架构设计.md 7.2.26节）
- 实现 IAdminService（知识源管理、AI配置管理，通过Feign调用chat-service）
- 实现 KnowledgeController、AiConfigController（路由 `/admin/knowledge/**`、`/admin/ai/**`）

**第10周**：
- 实现统计服务（对话统计、热门问题、用户统计、API调用统计、回答满意度统计）
- 实现 AI 调用追踪和评估（基于 `sys_ai_operation_log` 表 + Spring AI Micrometer，**不使用 LangSmith**，详见 09-部署方案.md 9.9.3 节）
- 实现工具管理（ToolController，启用/禁用/测试工具，启用状态通过 ArgumentServiceApi 存储在 sys_argument 表，参数配置存储在 Nacos）
- 实现召回测试（`/admin/knowledge/retrieve-test`，验证RAG检索效果和引用来源分组）
- 实现系统健康状态检测（`/admin/system/health`，检测MySQL/Redis/Nacos/Milvus/LLM 5个组件）
- **复用脚手架现有用户管理功能**（sys_user 表、admin-service 已实现，无需重复开发）
- 验证B端完整功能

---

#### Phase 6：测试与优化（第11-12周）

**第11周**：
- 执行黄金测试问题集（38题）
- 分析测试结果，定位问题
- 修复发现的Bug

**第12周**：
- 性能优化（Embedding缓存、检索参数调优）
- 代码质量检查
- 最终验证所有功能
- 编写部署文档

## 10.3 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|-----|------|------|---------|
| Milvus部署复杂 | 中 | 高 | 使用docker-compose一键部署，提供详细部署文档 |
| 向量检索效果不佳 | 中 | 中 | 优化分块策略、调整检索参数、增加元数据过滤 |
| 工具调用安全性 | 低 | 高 | 限制文件读取路径白名单、参数校验、禁止访问敏感目录 |
| LLM成本控制 | 中 | 中 | 缓存Embedding结果、优化Prompt、限制响应长度 |
| 知识源更新不及时 | 低 | 中 | 提供手动触发更新接口、定时自动更新 |
| 大文件读取内存溢出 | 低 | 中 | 限制单文件最大读取大小（10MB）、流式读取 |

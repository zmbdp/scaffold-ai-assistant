# B 端前端接口文档 - 总说明

> 本文档为 B 端（管理端）前端开发的公共约定，所有 B 端功能模块的分文档均遵循此约定。
>
> **适用对象**：B 端管理系统（userFrom=sys 的系统用户），包括知识库管理、AI 配置、工具管理、统计分析、操作日志、反馈管理、系统健康、用户管理、字典管理、参数管理等模块。
>
> **不包含**：地图服务接口（独立模块，不在本文档范围）。

---

## 一、网关地址

```
http://localhost:10030
```

所有接口都经过网关（zmbdp-gateway-service）统一入口，前端只需请求「网关地址 + 接口路径」。

## 二、路由前缀与路径映射

B 端接口统一前缀 `/admin`。网关配置 `Path=/admin/**` + `StripPrefix=1`，即网关会剥离第一段 `/admin`，转发到 admin-service（AI 相关接口会由 admin-service 再通过 Feign 转发到 chat-service）。

| 前端请求 | 网关转发目标 | 说明 |
| --- | --- | --- |
| `/admin/knowledge/sources` | admin-service `/knowledge/sources` | 知识库管理 |
| `/admin/ai/config` | admin-service `/ai/config` → Feign → chat-service | AI 配置 |
| `/admin/tools` | admin-service `/tools` → Feign → chat-service | 工具管理 |
| `/admin/statistics/*` | admin-service `/statistics/*` → Feign → chat-service | 统计分析 |
| `/admin/sys_user/*` | admin-service `/sys_user/*` | B 端用户 |
| `/admin/app_user/*` | admin-service `/app_user/*` | C 端用户管理 |
| `/admin/dictionary/*` | admin-service `/dictionary/*` | 字典管理 |
| `/admin/argument/*` | admin-service `/argument/*` | 参数管理 |
| `/admin/feedback/list` | admin-service `/feedback/list` → Feign → chat-service | 反馈管理 |
| `/admin/system/health` | admin-service `/system/health` → Feign → chat-service | 系统健康 |
| `/admin/knowledge/logs/list` | admin-service `/knowledge/logs/list` → Feign → chat-service | 操作日志 |

> **注意**：前端始终请求 `/admin/xxx`，无需关心后端 Feign 转发细节。

## 三、JWT 认证机制

### 3.1 请求头格式（所有需认证的接口必填）

```
Authorization: Bearer {accessToken}
Content-Type: application/json
```

> **注意**：`Bearer` 后必须有一个空格，再接 token 值。token 来自 B 端登录接口返回的 `accessToken` 字段。

### 3.2 B 端用户身份

B 端用户的 JWT 中 `userFrom=sys`。网关 `AuthFilter` 会阻止 `userFrom=app`（C 端用户）的 token 访问 `/admin/**` 路径，返回 `401004 令牌验证失败！`。

**测试 B 端接口时务必使用 B 端登录（`/admin/sys_user/login/password`）获取的 token，切勿复用 C 端 `/portal/login/**` 产出的 token。**

### 3.3 网关白名单（以下路径不需要 token）

- `/**/register/**`
- `/admin/codeLogin`
- `/**/login/**`
- `/**/send_code/**`
- `/**/nologin/**`
- `/**/test/**`

> B 端登录接口 `/admin/sys_user/login/password` 匹配 `/**/login/**`，因此不需要 token。

### 3.4 网关校验流程

不在白名单的请求，网关 `AuthFilter` 依次校验：

1. token 是否存在（缺失返回 `401000 令牌不能为空`，HTTP 401）
2. token 是否有效（无效返回 `401001 令牌已过期或验证不正确！`，HTTP 401）
3. Redis 登录态是否存在（过期返回 `401003 登录状态已过期！`，HTTP 401）
4. `/admin/**` 路径校验 `userFrom=sys`（不匹配返回 `401004 令牌验证失败！`，HTTP 401）

> **重要**：网关鉴权失败时，HTTP 状态码为 401/403/500，响应体为 `Result` JSON 结构（含 `code`/`errMsg`）。前端应同时处理 HTTP 状态码和业务 code。

## 四、统一响应结构 `Result<T>`

除特殊说明外，所有接口返回统一 JSON 结构：

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `code` | int | 响应码，成功固定 `200000` |
| `errMsg` | String | 消息描述，成功为 `"操作成功"` |
| `data` | T | 业务数据，无数据时为 `null` |

> **前端判断成功**：`code === 200000`。不要仅用 HTTP 状态码判断（业务错误统一返回 HTTP 200 + 非 200000 的 code；网关鉴权错误返回 HTTP 401/403 + Result JSON）。

## 五、分页响应结构 `BasePageVO<T>`

分页接口的 `data` 字段结构：

```json
{
  "totals": 100,
  "totalPages": 5,
  "list": []
}
```

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `totals` | Integer | 总记录数 |
| `totalPages` | Integer | 总页数 |
| `list` | List&lt;T&gt; | 数据列表 |

### 分页请求参数约定

B 端分页接口统一使用 `pageNo` / `pageSize` 两个查询参数：

| 参数 | 类型 | 默认值 | 含义 |
| --- | --- | --- | --- |
| `pageNo` | Integer | `1` | 页码，从 1 开始 |
| `pageSize` | Integer | 接口而定（常见 10/20） | 每页大小 |

> **注意**：部分列表查询接口（如 B 端用户列表 `POST /admin/sys_user/list`、C 端用户查询 `POST /admin/app_user/list/search`）使用 POST + 请求体 DTO 传参，分页字段在请求体内（`pageNo`/`pageSize`）。具体见各模块文档。

## 六、日期与时间格式约定

| 类型 | 格式 | 示例 | Java 类型 |
| --- | --- | --- | --- |
| 日期参数 | BIGINT，`YYYYMMDD` | `20260723` | Long |
| 日期字段（createDate/updateDate） | BIGINT，`YYYYMMDD` | `20260723` | Long |
| 精确时间字段（createTime） | `yyyy-MM-dd HH:mm:ss` 字符串 | `2026-07-23 15:30:00` | LocalDateTime 序列化为字符串 |
| 时间戳字段（timestamp/lastUsedTime） | 毫秒级时间戳 | `1784900000000` | Long |

> **过滤参数**：`startDate` / `endDate` 统一使用 BIGINT `YYYYMMDD` 格式（Long 类型），如 `20260701`。

| --- | --- | --- | --- |

| `400006` | 参数类型不匹配 | 参数类型错误 | 检查入参类型 |
| `400020` | 请求过于频繁，请稍后重试 | 触发限流 | 提示稍后重试 |
| `400007` | 账号已停用，登录失败 | B 端用户被停用 | 提示账号已停用 |
| `401000` | 令牌不能为空 | 缺 token（HTTP 401） | 跳转登录 |
| `401001` | 令牌已过期或验证不正确！ | token 无效（HTTP 401） | 跳转登录 |
| `401003` | 登录状态已过期！ | Redis 登录态过期（HTTP 401） | 跳转登录 |

| `404001` | url 未找到 | 接口路径不存在（HTTP 404） | 检查请求路径 |

| `500001` | 操作失败 | 业务操作失败 | 提示失败原因 |

## 八、操作审计说明

B 端管理操作（新增/更新/删除知识源、删除文档、触发同步、更新 AI 配置、更新工具配置、上传文档等）通过 `@LogAction` 注解自动记录到 `operation_log` 表（注意：不是 `sys_ai_operation_log` 表）。

- `operation_log` 表：记录 B 端管理操作审计（知识源 CRUD、配置修改等）
- `sys_ai_operation_log` 表：记录 AI 调用链路日志（对话、检索、向量嵌入等 AI 编排层埋点）

> B 端「操作日志」查询接口（`GET /admin/knowledge/logs/list`）查询的是 `sys_ai_operation_log` 表，即 AI 调用链路日志，不是管理操作审计。

### 9.2 文档状态（status 字段）

### 9.10 Agent 工具名称（用于 `PUT /admin/tools/{name}` 和 `POST /admin/tools/{name}/test`）

| 工具 name | 含义 |
| --- | --- |
| `readFile` | 读取指定文件内容 |
| `searchCode` | 搜索代码中的类/方法 |
| `listDir` | 列出目录结构 |
| `searchInFile` | 在文件中搜索关键字 |
| `nacosConfig` | 查询 Nacos 配置项 |

## 十二、前端集成注意事项

### 12.1 Token 使用

- B 端登录返回 `accessToken` 后，所有需鉴权接口在请求头携带 `Authorization: Bearer {accessToken}`
- `Bearer` 后必须有一个空格
- token 过期或登录态失效时（`401xxx` 错误码）跳转登录页

### 12.2 路径前缀

- 所有 B 端接口统一加 `/admin` 前缀
- 文件上传接口 `/file/upload` 无 `/admin` 前缀（通用接口）

### 12.3 权限控制

- B 端接口（`/admin/**`）仅允许 `userFrom=sys` 的 token 访问
- B 端用户身份（`identity`）区分权限：`super_admin` 可管理用户，`platform_admin` 不可
- 前端应根据登录信息接口返回的 `identity` 字段控制 UI 元素显示（如「新增用户」按钮仅 `super_admin` 可见）

### 12.4 日期参数

- 所有日期过滤参数（`startDate`/`endDate`/`days`）使用 BIGINT `YYYYMMDD` 格式
- `days` 参数为 Integer 类型，表示统计天数窗口

### 12.5 操作审计与 AI 日志区分

- B 端管理操作审计记录在 `operation_log` 表（通过 `@LogAction` 自动记录），当前无查询接口
- AI 调用链路日志记录在 `sys_ai_operation_log` 表，通过 `GET /admin/knowledge/logs/list` 查询

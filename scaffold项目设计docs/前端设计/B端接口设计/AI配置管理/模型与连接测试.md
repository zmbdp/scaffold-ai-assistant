# 模型与连接测试

> 模块：可用模型列表查询、AI 连接测试
>
> 公共约定（网关地址、JWT 认证、统一响应结构 `Result<T>`、错误码、操作审计）见 `总说明.md`，本文档不再重复。

## 功能介绍

本模块用于查看 chat-service 已注册的可用模型，并验证 AI 模型服务的连通性，包含两个接口：

- **获取可用模型列表**：返回 Nacos `spring.ai.models` 配置中注册的全部模型信息（名称、提供商、类型、能力、启用状态等），供前端展示模型清单。
- **测试 AI 连接**：使用默认模型向 AI 服务发起一次轻量对话调用，验证 API Key、网络、模型服务是否正常。无入参，失败时返回明确错误信息。

**调用链路**：前端 `GET /admin/ai/models`、`POST /admin/ai/test` → 网关 StripPrefix=1 → admin-service `AiConfigController` → Feign `AiConfigApi` → chat-service `AiConfigController` → `AdminServiceImpl` → `ModelServiceImpl`。

**模型配置来源**：模型清单由 chat-service 启动时从 Nacos `spring.ai.models` 列表加载到 `ModelProperties`，并为每个 `enabled=true` 的模型构建 `ChatClient` 实例。Nacos 配置变更时通过 `@RefreshScope` 自动重建。

## 一、获取可用模型列表

### 1.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `GET /admin/ai/models` |
| 认证 | 需要 B 端 token（`userFrom=sys`） |
| 操作审计 | 无（查询接口不记录 `@LogAction`） |
| Content-Type | 无需请求体 |
| 权限 | B 端登录用户均可访问 |

### 1.2 请求参数

无。

### 1.3 请求示例

```http
GET /admin/ai/models HTTP/1.1
Authorization: Bearer {accessToken}
```

### 1.4 响应结构

`Result<List<ModelVO>>`，`data` 为 `ModelVO` 数组。`ModelVO` 字段：

| 字段 | 类型 | 含义 | 说明 |
| --- | --- | --- | --- |
| `name` | String | 模型名称 | 如 `deepseek-v4-flash`、`qwen-vl-plus` |
| `provider` | String | 模型提供商 | 对应 `ModelProvider` 枚举，实际值为 Nacos 配置中的原始字符串 |
| `type` | String | 模型类型 | 对应 `ModelType` 枚举：`TEXT_ONLY` / `TEXT_AND_IMAGE` / `EMBEDDING` |
| `capabilities` | List&lt;String&gt; | 模型能力列表 | 对应 `ModelCapability` 枚举：`TEXT` / `IMAGE` / `EMBEDDING` / `FUNCTION_CALL` |
| `enabled` | Boolean | 是否启用 | `true` 表示该模型已注册到 ChatClient 实例池可用 |
| `description` | String | 模型描述 | **当前后端未填充**，恒为 `null`（模型配置源 `ModelConfig` 无此字段，仅作预留） |

> **数据来源说明**：列表由 `ModelServiceImpl.listModels()` 将 `ModelConfig` 列表通过 `BeanCopyUtil.copyListProperties` 拷贝为 `ModelVO` 列表。返回的是 Nacos `spring.ai.models` 中配置的**全部模型**（含 `enabled=false` 的未启用模型），前端可据此区分启用/未启用状态。

### 1.5 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": [
    {
      "name": "deepseek-v4-flash",
      "provider": "DASHSCOPE",
      "type": "TEXT_ONLY",
      "capabilities": ["TEXT", "FUNCTION_CALL"],
      "enabled": true,
      "description": null
    },
    {
      "name": "qwen-vl-plus",
      "provider": "DASHSCOPE",
      "type": "TEXT_AND_IMAGE",
      "capabilities": ["TEXT", "IMAGE"],
      "enabled": true,
      "description": null
    },
    {
      "name": "text-embedding-v1",
      "provider": "DASHSCOPE",
      "type": "EMBEDDING",
      "capabilities": ["EMBEDDING"],
      "enabled": true,
      "description": null
    }
  ]
}
```

### 1.6 错误码说明

| code | errMsg | 触发场景 |
| --- | --- | --- |
| `200000` | 操作成功 | 正常返回 |
| `500000` | 获取模型列表失败 / 服务繁忙请稍后重试 | Feign 调用 chat-service 失败，或 chat-service 内部异常 |

> 本接口不触发 AI 模块专属错误码。Nacos 未配置 `spring.ai.models` 时返回空数组 `[]`（不报错）。

### 1.7 枚举值

#### 模型类型 `type`（`ModelType` 枚举）

| 值 | 含义 |
| --- | --- |
| `TEXT_ONLY` | 纯文本模型（仅支持文本对话） |
| `TEXT_AND_IMAGE` | 图文模型（支持文本 + 图片对话） |
| `EMBEDDING` | 嵌入模型（用于向量生成） |

#### 模型能力 `capabilities` 元素（`ModelCapability` 枚举）

| 值 | 含义 |
| --- | --- |
| `TEXT` | 文本生成 |
| `IMAGE` | 图片理解 |
| `EMBEDDING` | 向量嵌入 |
| `FUNCTION_CALL` | 函数调用（工具调用） |

#### 模型提供商 `provider`（`ModelProvider` 枚举）

| 值 | 含义 |
| --- | --- |
| `DASHSCOPE` | 通义千问（阿里云 DashScope） |
| `OPENAI` | OpenAI |
| `ZHIPU` | 智谱 AI |
| `GEMINI` | Google Gemini |

> v1.0 仅实现 `DASHSCOPE`（通义千问）。`provider` / `type` / `capabilities` 字段均为字符串/字符串数组，实际返回值取决于 Nacos `spring.ai.models` 配置中填写的原始值。

### 1.8 前端注意事项

- `description` 字段恒为 `null`，不要依赖它做展示；如需模型说明，前端可维护本地映射或展示 `name`。
- 列表包含**未启用**（`enabled=false`）的模型，前端可用 `enabled` 字段做状态标记。
- 默认文本/视觉模型的判定规则：`defaultModel=true` 且 `type=TEXT_ONLY` 为默认文本模型；`defaultModel=true` 且 `type=TEXT_AND_IMAGE` 为默认视觉模型（该标志在 `ModelConfig` 中，未透出到 `ModelVO`，前端无法直接获知默认模型，需通过 `GET /admin/ai/config` 的 `defaultModel` / `defaultVisionModel` 字段获取）。
- 配置变更需重启或触发 Nacos 热刷新后，再次查询才能看到新模型。

## 二、测试 AI 连接

### 2.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `POST /admin/ai/test` |
| 认证 | 需要 B 端 token（`userFrom=sys`） |
| 操作审计 | 无（`@LogAction` 未标注） |
| Content-Type | 无需请求体 |
| 权限 | B 端登录用户均可访问 |

### 2.2 请求参数

无（无路径参数、无查询参数、无请求体）。

> 后端使用**默认文本模型**和**默认测试消息** `"你好，请回复'连接正常'"` 发起一次 ChatClient 调用。前端无法指定模型或消息内容。

### 2.3 请求示例

```http
POST /admin/ai/test HTTP/1.1
Authorization: Bearer {accessToken}
```

### 2.4 响应结构

`Result<Void>`，无 `data`。

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `code` | int | 响应码，成功为 `200000` |
| `errMsg` | String | 消息描述，成功为 `"操作成功"`；失败为 `"AI 连接测试失败"` |
| `data` | null | 无业务数据 |

**成功与失败的判定**（chat-service 内部逻辑）：

- 调用 `ChatClient.prompt().user(testMessage).call().chatResponse()` 发起一次同步对话。
- 若返回的 `ChatResponse` 非空、且 `getResult().getOutput().getText()` 非空，判定为**成功**，返回 `code=200000`。
- 若调用过程抛异常（API Key 缺失/无效、网络不通、模型服务不可达、超时等），或返回内容为空，判定为**失败**，返回 `code=500000`、`errMsg="AI 连接测试失败"`。

### 2.5 响应示例

成功：

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": null
}
```

失败：

```json
{
  "code": 500000,
  "errMsg": "AI 连接测试失败",
  "data": null
}
```

### 2.6 错误码说明

| code | errMsg | 触发场景 | HTTP 状态码 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | AI 服务连通正常，返回了非空响应内容 | 200 |
| `500000` | AI 连接测试失败 | API Key 缺失/无效、网络不通、模型服务不可达、响应内容为空等导致调用失败 | 500 |
| `500000` | 服务繁忙请稍后重试 | admin-service 与 chat-service 之间 Feign 调用本身失败（非业务失败） | 500 |

> **关于 AI 模块专属错误码的说明**：`ResultCode` 中定义了 `500022 AI 配置未设置`、`500023 AI 服务连接失败`、`500024 AI 服务响应超时`，但本测试连接接口**不会**抛出这些码。chat-service 的 `testAiConnection` 方法将所有异常捕获并返回 `false`，由 Controller 统一转为 `code=500000` + `"AI 连接测试失败"`。因此前端只需按 `200000`（成功）/非 `200000`（失败）判断即可，**不要**针对 `500022`/`500023`/`500024` 做特殊处理。
>
> `500023 AI 服务连接失败` 实际仅在 C 端流式对话（`ChatServiceImpl`）中使用，不用于 B 端测试连接。

### 2.7 枚举值

本接口无请求参数，无枚举。

### 2.8 前端注意事项

- 本接口无入参，前端点击「测试连接」按钮直接发起 `POST /admin/ai/test` 即可。
- 失败时 `errMsg` 固定为 `"AI 连接测试失败"`，**不包含具体原因**（如 API Key 无效、超时等细节仅记录在后端日志）。前端只需提示「连接失败，请检查 API Key 与网络配置」即可，无法从响应中区分具体失败类型。
- 测试使用默认文本模型，不测试视觉模型和 Embedding 模型。
- 该接口为同步调用，AI 模型响应可能较慢，前端应设置合理的请求超时（建议 ≥ 30 秒）并给按钮加 loading 状态，避免重复点击。
- 连接失败常见排查方向：Nacos 中 `spring.ai.dashscope.api-key` 是否配置、API Key 是否有效、网络是否能访问模型服务、默认文本模型是否启用。

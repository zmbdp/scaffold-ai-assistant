# 对话统计

## 功能介绍

对话统计接口用于在 B 端管理后台展示对话总量概览，包括对话总数、今日对话数、平均响应时间、活跃用户数以及近 7 天每日对话量趋势。

数据来源：`sys_ai_conversation` 表，仅统计 `is_deleted=0` 且 `status='SUCCESS'` 的有效对话记录。

接口走 Redis 缓存（TTL=300 秒），缓存 key 为 `stats:conversation`，无参数因此无需考虑参数隔离。

---

## 接口详情

### 获取对话统计

| 项 | 值 |
| --- | --- |
| 路径 | `GET /admin/statistics/conversations` |
| 认证 | 必须携带 B 端 JWT（`userFrom=sys`），见总说明第三章 |
| Content-Type | `application/json` |
| 请求参数 | 无 |

#### 调用链路

前端请求 `/admin/statistics/conversations` → 网关 StripPrefix=1 → admin-service `StatisticsController#getConversationStatistics()` → Feign 调用 chat-service `StatisticsApi#getConversationStatistics()` → `StatisticsServiceImpl#getConversationStats()`。

---

### 响应结构

返回 `Result<ConversationStatisticsVO>`，`data` 字段为对话统计聚合 VO。

| 字段 | 类型 | 含义 | 数据来源 / 计算口径 |
| --- | --- | --- | --- |
| `totalConversations` | Long | 对话总数 | `COUNT(*)` WHERE `is_deleted=0 AND status='SUCCESS'` |
| `todayConversations` | Long | 今日对话数 | `COUNT(*)` WHERE `is_deleted=0 AND status='SUCCESS' AND create_date={今日 YYYYMMDD}` |
| `avgResponseTime` | Long | 平均响应时间（毫秒） | `IFNULL(AVG(response_time), 0)` WHERE `is_deleted=0 AND status='SUCCESS'` |
| `activeUsers` | Long | 活跃用户数（近 7 天） | `COUNT(DISTINCT user_id)` WHERE `is_deleted=0 AND status='SUCCESS' AND create_date >= 近7天起始日` |
| `trendData` | List&lt;TrendData&gt; | 趋势数据（近 7 天） | 按日期聚合的对话量，补全无对话日期为 0 |

**TrendData 字段**：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `date` | String | 日期，格式 `yyyy-MM-dd`（如 `2026-07-23`） |
| `count` | Long | 当日对话量 |

> **趋势补全逻辑**：Service 层会补全近 7 天中无对话记录的日期（`count=0`），确保趋势图连续。日期顺序按时间正序排列（最早 → 最新）。

---

### 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "totalConversations": 1280,
    "todayConversations": 56,
    "avgResponseTime": 3200,
    "activeUsers": 23,
    "trendData": [
      { "date": "2026-07-17", "count": 120 },
      { "date": "2026-07-18", "count": 156 },
      { "date": "2026-07-19", "count": 198 },
      { "date": "2026-07-20", "count": 145 },
      { "date": "2026-07-21", "count": 210 },
      { "date": "2026-07-22", "count": 395 },
      { "date": "2026-07-23", "count": 56 }
    ]
  }
}
```

> **空数据场景**：无任何对话记录时，`totalConversations=0`、`todayConversations=0`、`avgResponseTime=0`、`activeUsers=0`、`trendData` 仍返回近 7 天完整日期列表（每日 `count=0`）。

---

### 错误码说明

| code | errMsg | 含义 | 场景 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | 成功 | — |
| `401000` | 令牌不能为空 | 缺 token（HTTP 401） | 未携带 Authorization |
| `401003` | 登录状态已过期！ | Redis 登录态过期（HTTP 401） | token 过期 |
| `401004` | 令牌验证失败！ | 越权访问（HTTP 401） | `userFrom=app` 的 C 端 token 访问 `/admin/**` |
| `404000` | 服务未找到 | chat-service 不可用（HTTP 404） | chat-service 未注册到 Nacos |
| `500000` | 服务繁忙请稍后重试 | 服务异常（HTTP 500） | Feign 调用异常或数据库异常 |

> 本接口为只读查询，不涉及业务参数校验错误码（无 `400000` 场景）。当 chat-service Feign 调用返回非 `200000` 时，admin-service 会抛出 `ServiceException("获取对话统计失败")`，HTTP 200 + `code=500001`。

---

## 前端注意事项

1. **缓存说明**：本接口走 Redis 缓存，TTL=300 秒（5 分钟）。数据更新有最长 5 分钟延迟，新产生的对话不会立即反映在统计中。无需前端额外处理缓存。

2. **统计口径**：仅统计 `status='SUCCESS'`（调用成功）且未软删除（`is_deleted=0`）的对话。失败的对话（FAILED/TIMEOUT）不计入总量与趋势。

3. **活跃用户数固定 7 天**：本接口的 `activeUsers` 固定统计近 7 天活跃用户（Service 常量 `ACTIVE_USER_DAYS=7`），不支持自定义天数。如需自定义天数窗口，请使用「用户统计」接口（`GET /admin/statistics/users?days=`）。

4. **趋势数据固定 7 天**：`trendData` 固定返回近 7 天数据（Service 常量 `TREND_DAYS=7`），无参数可调。

5. **avgResponseTime 单位**：毫秒（ms），来自 `sys_ai_conversation.response_time` 字段。

6. **无审计/限流注解**：本接口为只读查询，未标注 `@LogAction`（不写入管理审计）和 `@RateLimit`（不限流）。

# 热门问题

## 功能介绍

热门问题接口用于在 B 端管理后台展示用户提问频次最高的 N 个问题，帮助管理员了解用户关注热点。

数据来源：`sys_ai_conversation` 表，按 `question` 字段分组统计（过滤 `is_deleted=0`、`status='SUCCESS'` 且 `question` 非空），按被问次数降序排序，取前 N 条。

接口走 Redis 缓存（TTL=300 秒）。

---

## 接口详情

### 获取热门问题

| 项 | 值 |
| --- | --- |
| 路径 | `GET /admin/statistics/questions` |
| 认证 | 必须携带 B 端 JWT（`userFrom=sys`），见总说明第三章 |
| Content-Type | `application/json` |

#### 调用链路

前端请求 `/admin/statistics/questions` → 网关 StripPrefix=1 → admin-service `StatisticsController#getHotQuestions(limit)` → Feign 调用 chat-service `StatisticsApi#getHotQuestions(limit)` → `StatisticsServiceImpl#getTopQuestions(limit)`。

---

### 请求参数

| 参数名 | 类型 | 必传 | 默认值 | 校验规则 | 含义 |
| --- | --- | --- | --- | --- | --- |
| `limit` | Integer | 否 | `10` | Service 层兜底：为 `null` 或 `<= 0` 时取默认值 `10` | 返回热门问题数量（前 N 条） |

> **兜底逻辑**：chat-service 端 `StatisticsServiceImpl#getTopQuestions` 对参数做了二次兜底——若 `limit <= 0`，强制取 `10`。因此前端传 `0`、负数或不传，结果均为返回前 10 条。

---

### 请求示例

**示例 1：使用默认 limit（不传参）**

```
GET /admin/statistics/questions
Authorization: Bearer {accessToken}
```

**示例 2：指定 limit**

```
GET /admin/statistics/questions?limit=20
Authorization: Bearer {accessToken}
```

---

### 响应结构

返回 `Result<List<HotQuestionVO>>`，`data` 为热门问题 VO 列表（按 `count` 降序）。

**HotQuestionVO 字段**：

| 字段 | 类型 | 含义 | 数据来源 |
| --- | --- | --- | --- |
| `question` | String | 问题内容 | `sys_ai_conversation.question` |
| `count` | Integer | 被问次数 | `COUNT(*)` 分组统计 |
| `lastAskedTime` | Long | 最后一次提问时间（毫秒时间戳） | `UNIX_TIMESTAMP(MAX(create_time)) * 1000` |

> **时间戳说明**：`lastAskedTime` 为毫秒级时间戳（Long），由 SQL `UNIX_TIMESTAMP(MAX(create_time)) * 1000` 转换得到。前端可使用 `new Date(lastAskedTime)` 直接构造日期对象。

---

---

### 错误码说明

| code | errMsg | 含义 | 场景 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | 成功 | — |
| `401000` | 令牌不能为空 | 缺 token（HTTP 401） | 未携带 Authorization |
| `401003` | 登录状态已过期！ | Redis 登录态过期（HTTP 401） | token 过期 |
| `401004` | 令牌验证失败！ | 越权访问（HTTP 401） | C 端 token 访问 `/admin/**` |
| `404000` | 服务未找到 | chat-service 不可用（HTTP 404） | chat-service 未注册到 Nacos |
| `500000` | 服务繁忙请稍后重试 | 服务异常（HTTP 500） | Feign 调用异常或数据库异常 |

> 当 chat-service Feign 调用返回非 `200000` 时，admin-service 抛出 `ServiceException("获取热门问题失败")`，HTTP 200 + `code=500001`。

---

## 前端注意事项

1. **统计口径**：仅统计 `status='SUCCESS'`、`is_deleted=0` 且 `question` 非空（`IS NOT NULL AND != ''`）的对话。相同 `question` 文本完全一致才归为一组（精确匹配，不做模糊归并）。

2. **缓存说明**：本接口走 Redis 缓存，TTL=300 秒（5 分钟），缓存 key 为 `stats:questions`。

   > **缓存 key 不含 limit 参数**：当前实现的缓存 key 固定为 `stats:questions`，未拼接 `limit` 参数。这意味着首次以 `limit=10` 请求并写入缓存后，5 分钟内再用 `limit=20` 请求会命中缓存并返回前 10 条（而非前 20 条）。如需切换 limit 立即生效，建议后端评估是否将 limit 纳入缓存 key；前端当前可视为「不同 limit 在 5 分钟窗口内可能返回相同结果集」。

3. **排序规则**：按 `count DESC`（被问次数降序）。次数相同时的次级排序由数据库默认行为决定（SQL 未显式指定次级排序）。

4. **limit 兜底**：传入 `null`、`0`、负数均会被 Service 层兜底为 `10`，不会报参数错误。

5. **无审计/限流注解**：本接口为只读查询，未标注 `@LogAction` 和 `@RateLimit`。

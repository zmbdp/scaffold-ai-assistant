# 满意度统计

## 功能介绍

回答满意度统计接口用于在 B 端管理后台展示用户对 AI 回答的反馈聚合情况，包括点赞率、点踩率、反馈率以及点踩原因分布，帮助管理员评估 AI 回答质量并定位问题（如 `CODE_ERROR` 占比高说明知识库代码分块不准确，需优化分块策略）。

数据来源：`sys_ai_feedback` 表（反馈数据）+ `sys_ai_conversation` 表（对话总数，用于计算反馈率）。

**`startDate`/`endDate` 过滤参数已生效**，支持按日期范围过滤反馈统计。接口走 Redis 缓存（TTL=300 秒），**缓存 key 包含日期参数**（`stats:feedback:{startDate}:{endDate}`），不同日期范围的统计结果互相隔离。

---

## 接口详情

### 获取回答满意度统计

| 项 | 值 |
| --- | --- |
| 路径 | `GET /admin/statistics/feedback` |
| 认证 | 必须携带 B 端 JWT（`userFrom=sys`），见总说明第三章 |
| Content-Type | `application/json` |

#### 调用链路

前端请求 `/admin/statistics/feedback` → 网关 StripPrefix=1 → admin-service `StatisticsController#getFeedbackStatistics(startDate, endDate)` → Feign 调用 chat-service `StatisticsApi#getFeedbackStatistics(startDate, endDate)` → `StatisticsServiceImpl#getFeedbackStats(startDate, endDate)`。

---

### 请求参数

| 参数名 | 类型 | 必传 | 默认值 | 格式 | 含义 |
| --- | --- | --- | --- | --- | --- |
| `startDate` | Long | 否 | 不传则不加起始日期过滤 | BIGINT `YYYYMMDD`（如 `20260701`） | 起始日期，过滤 `sys_ai_feedback.create_date >= startDate` |
| `endDate` | Long | 否 | 不传则不加结束日期过滤 | BIGINT `YYYYMMDD`（如 `20260723`） | 结束日期，过滤 `sys_ai_feedback.create_date <= endDate` |

> **过滤参数生效说明**：`startDate`/`endDate` 均为可选参数，分别独立生效。
> - 仅传 `startDate`：过滤 `create_date >= startDate`（统计该日期至今的反馈）。
> - 仅传 `endDate`：过滤 `create_date <= endDate`（统计该日期及之前的反馈）。
> - 两者均传：过滤 `create_date BETWEEN startDate AND endDate`（闭区间）。
> - **两者均不传：不加日期过滤，返回全量反馈统计**（不是默认近 7 天）。

> **日期格式**：`startDate`/`endDate` 为 `Long` 类型的 BIGINT，格式 `YYYYMMDD`（如 `20260701`），与脚手架统一的日期参数约定一致。前端传入字符串形式的日期会被 Spring 解析为 Long；建议前端直接传数字。

> **重要口径说明（关于 feedbackRate 分母）**：`totalConversations`（反馈率分母）由 `countTotalConversationsForFeedback` 查询，SQL 为 `SELECT COUNT(*) FROM sys_ai_conversation WHERE is_deleted=0 AND status='SUCCESS'`，**该查询不带任何参数，不受 `startDate`/`endDate` 影响**，始终返回全量有效对话总数。因此 `feedbackRate` 的分母始终是全量对话数，即使指定了日期范围也是如此。前端解读反馈率时需注意此口径。

---

### 请求示例

**示例 1：查询全量反馈统计（不传日期参数）**

```
GET /admin/statistics/feedback
Authorization: Bearer {accessToken}
```

**示例 2：按日期范围过滤**

```
GET /admin/statistics/feedback?startDate=20260701&endDate=20260723
Authorization: Bearer {accessToken}
```

**示例 3：仅指定起始日期**

```
GET /admin/statistics/feedback?startDate=20260701
Authorization: Bearer {accessToken}
```

### 响应结构

返回 `Result<FeedbackStatisticsVO>`，`data` 为回答满意度统计聚合 VO。

**FeedbackStatisticsVO 字段**：

| 字段 | 类型 | 含义 | 数据来源 / 计算口径 |
| --- | --- | --- | --- |
| `likeCount` | Long | 点赞数 | `COUNT(*)` WHERE `feedback_type='LIKE'`（+可选日期过滤） |
| `dislikeCount` | Long | 点踩数 | `COUNT(*)` WHERE `feedback_type='DISLIKE'`（+可选日期过滤） |
| `totalFeedback` | Long | 反馈总数（`likeCount + dislikeCount`） | `COUNT(*)` FROM `sys_ai_feedback`（+可选日期过滤） |
| `feedbackCount` | Long | 反馈用户数（按 `user_id` 去重） | `COUNT(DISTINCT user_id)`（+可选日期过滤） |
| `totalConversations` | Long | 对话总数（反馈率分母） | `COUNT(*)` FROM `sys_ai_conversation` WHERE `is_deleted=0 AND status='SUCCESS'`（**不受日期过滤影响**） |
| `likeRate` | Double | 点赞率（%） | `likeCount / totalFeedback × 100`，保留两位小数；反馈总数为 0 时返回 `0.0` |
| `dislikeRate` | Double | 点踩率（%） | `dislikeCount / totalFeedback × 100`，保留两位小数；反馈总数为 0 时返回 `0.0` |
| `feedbackRate` | Double | 反馈率（%） | `totalFeedback / totalConversations × 100`，保留两位小数；对话总数为 0 时返回 `0.0` |
| `dislikeReasonDistribution` | List&lt;DislikeReasonDistribution&gt; | 点踩原因分布 | 按点踩原因分组统计（+可选日期过滤） |

**DislikeReasonDistribution 字段**：

| 字段 | 类型 | 含义 | 数据来源 |
| --- | --- | --- | --- |
| `reason` | String | 点踩原因 | `sys_ai_feedback.dislike_reason`，枚举见下方 |
| `count` | Long | 该原因的点踩数 | `COUNT(*)` GROUP BY `dislike_reason` |
| `percentage` | Double | 该原因占比（%） | `count / dislikeCount × 100`，保留两位小数；点踩总数为 0 时返回 `0.0` |

> **比率计算精度**：所有比率均保留两位小数，采用 `Math.round(value * 10000.0) / 100.0` 实现（先放大 10000 倍四舍五入再除以 100，等价于保留两位小数）。
>
> **占比分母**：`percentage` 的分母为 `dislikeCount`（点踩总数），不是 `totalFeedback`（反馈总数）。即各原因占比之和为 100%（若所有点踩记录都有 `dislike_reason` 值）。

---

### 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "likeCount": 320,
    "dislikeCount": 80,
    "totalFeedback": 400,
    "feedbackCount": 180,
    "totalConversations": 1280,
    "likeRate": 80.0,
    "dislikeRate": 20.0,
    "feedbackRate": 31.25,
    "dislikeReasonDistribution": [
      { "reason": "CODE_ERROR", "count": 35, "percentage": 43.75 },
      { "reason": "OUTDATED", "count": 20, "percentage": 25.0 },
      { "reason": "IRRELEVANT", "count": 15, "percentage": 18.75 },
      { "reason": "OTHER", "count": 10, "percentage": 12.5 }
    ]
  }
}
```

> **空数据场景**：无任何反馈记录时，`likeCount=0`、`dislikeCount=0`、`totalFeedback=0`、`feedbackCount=0`、`totalConversations` 为当前有效对话总数、`likeRate=0.0`、`dislikeRate=0.0`、`feedbackRate=0.0`、`dislikeReasonDistribution` 返回空数组 `[]`。

> **dislikeReasonDistribution 排序**：按 `count DESC`（点踩数降序）排序。仅统计 `feedback_type='DISLIKE'` 且 `dislike_reason` 非空（`IS NOT NULL AND != ''`）的记录。

### 枚举值

#### feedbackType（反馈类型，本接口聚合维度）

| 值 | 含义 |
| --- | --- |
| `LIKE` | 点赞 |
| `DISLIKE` | 点踩 |

#### dislikeReason（点踩原因，分布项的 `reason` 字段）

| 值 | 含义 |
| --- | --- |
| `OUTDATED` | 信息过时 |
| `IRRELEVANT` | 答非所问 |
| `CODE_ERROR` | 代码错误 |
| `OTHER` | 其他 |

---

### 错误码说明

| code | errMsg | 含义 | 场景 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | 成功 | — |
| `401000` | 令牌不能为空 | 缺 token（HTTP 401） | 未携带 Authorization |
| `401003` | 登录状态已过期！ | Redis 登录态过期（HTTP 401） | token 过期 |
| `401004` | 令牌验证失败！ | 越权访问（HTTP 401） | C 端 token 访问 `/admin/**` |
| `404000` | 服务未找到 | chat-service 不可用（HTTP 404） | chat-service 未注册到 Nacos |
| `500000` | 服务繁忙请稍后重试 | 服务异常（HTTP 500） | Feign 调用异常或数据库异常 |

> 当 chat-service Feign 调用返回非 `200000` 时，admin-service 抛出 `ServiceException("获取回答满意度统计失败")`，HTTP 200 + `code=500001`。

## 前端注意事项

1. **startDate/endDate 过滤参数已生效**：两个参数独立可选，分别控制起始/结束日期边界。SQL 通过 `<if test="startDate != null">` 动态拼接，未传时该条件不生效。

2. **不传日期参数返回全量统计**：当 `startDate`/`endDate` 均不传时，**不加任何日期过滤，返回全量反馈统计**（所有历史反馈），不是默认近 7 天。
   > **源码澄清**：`FeedbackStatisticsVO` 类注释中提到「未传 startDate/endDate 时默认统计最近 7 天数据」，但实际 Service 实现（`StatisticsServiceImpl#getFeedbackStats`）与 Mapper XML 均未做默认 7 天处理，未传时返回全量。**以源码 SQL 行为为准**。

3. **缓存 key 含日期参数**：缓存 key 为 `stats:feedback:{startDate}:{endDate}`（如 `stats:feedback:20260701:20260723`、`stats:feedback:null:null`），不同日期范围的统计结果独立缓存、互不覆盖，切换日期范围可立即获取对应结果。

4. **feedbackRate 分母不受日期过滤影响**：`totalConversations` 始终为全量有效对话总数（`is_deleted=0 AND status='SUCCESS'`），不受 `startDate`/`endDate` 影响。因此按日期范围过滤后的 `feedbackRate` 含义为「该时间段内的反馈数 / 全量对话总数」，而非「该时间段内的反馈数 / 该时间段内的对话数」。前端展示反馈率时需注意此口径差异。

5. **dislikeReasonDistribution 分母为 dislikeCount**：`percentage` 的分母为 `dislikeCount`（点踩总数，已受日期过滤），各原因占比之和约等于 100%（若存在 `dislike_reason` 为空的点踩记录，则之和小于 100%）。

6. **日期格式**：`startDate`/`endDate` 为 `Long` 类型 BIGINT，格式 `YYYYMMDD`（如 `20260701`）。前端传参时确保为 8 位数字，不要带分隔符或时间部分。

7. **feedbackCount 含义**：`feedbackCount` 为反馈用户数（按 `user_id` 去重），不是反馈记录数。一个用户多次反馈只计一次。

8. **无审计/限流注解**：本接口为只读查询，未标注 `@LogAction` 和 `@RateLimit`。

# 工具统计

## 功能介绍

工具使用统计接口用于在 B 端管理后台展示 Agent 工具的调用情况，包括工具总调用次数以及各工具的调用次数、成功/失败次数等明细。

数据来源：`sys_ai_operation_log` 表的 `tool_calls` 字段（JSON 数组）。Service 层解析该 JSON 字段，按工具名称分组聚合统计。

> **tool_calls 字段已修复**：新对话中的工具调用会被正确统计。通过 `ToolCallRecorder` 包装 `ToolCallback` 实现，确保 Agent 工具调用链路被完整记录到 `sys_ai_operation_log.tool_calls` 字段。

接口走 Redis 缓存（TTL=300 秒），缓存 key 为 `stats:tools`。

---

## 接口详情

### 获取工具使用统计

| 项 | 值 |
| --- | --- |
| 路径 | `GET /admin/statistics/tools` |
| 认证 | 必须携带 B 端 JWT（`userFrom=sys`），见总说明第三章 |
| Content-Type | `application/json` |
| 请求参数 | 无 |

#### 调用链路

前端请求 `/admin/statistics/tools` → 网关 StripPrefix=1 → admin-service `StatisticsController#getToolStatistics()` → Feign 调用 chat-service `StatisticsApi#getToolStatistics()` → `StatisticsServiceImpl#getToolUsageStats()`。

---

### 数据来源与解析逻辑

1. **查询原始记录**：从 `sys_ai_operation_log` 表查询所有 `tool_calls` 非空、非 `''`、非 `'[]'` 的记录（SQL 仅查 `id` 和 `tool_calls` 两个字段）。
2. **解析 JSON**：`tool_calls` 字段为 JSON 数组，每项格式如下：

   ```json
   [
     {"name":"readFile","success":true,"duration":45,"summary":"读取成功"},
     {"name":"searchCode","success":false,"duration":120,"summary":"未找到匹配"}
   ]
   ```

3. **按工具名聚合**：按 `name` 分组统计调用次数、成功次数、失败次数，按调用次数降序排序。

---

### 响应结构

返回 `Result<ToolStatisticsVO>`，`data` 为工具使用统计聚合 VO。

**ToolStatisticsVO 字段**：

| 字段 | 类型 | 含义 | 数据来源 |
| --- | --- | --- | --- |
| `totalCalls` | Long | 工具总调用次数 | 解析所有 `tool_calls` JSON 后累加所有工具调用次数 |
| `toolUsage` | List&lt;ToolUsage&gt; | 工具使用详情列表（按 `callCount` 降序） | 按工具名分组聚合 |

**ToolUsage 字段**：

| 字段 | 类型 | 含义 | 数据来源 |
| --- | --- | --- | --- |
| `toolName` | String | 工具名称 | `tool_calls` JSON 中的 `name` 字段（如 `readFile`、`searchCode`） |
| `callCount` | Integer | 调用次数 | 按工具名分组 `COUNT` |
| `successCount` | Integer | 成功次数 | `success=true` 的次数 |
| `failCount` | Integer | 失败次数 | `success=false` 的次数 |
| `lastUsedTime` | Long | 最后使用时间（毫秒时间戳） | 当前实现固定为 `0`，详见下方注意事项 |

> **lastUsedTime 字段说明**：当前 `selectAllToolCallRecords` SQL 仅查询 `id` 和 `tool_calls` 两个字段，未返回 `create_time`，因此 Service 层无法计算真实最后使用时间，固定填 `0` 占位。如需展示工具最后使用时间，请参考「工具管理」接口（`GET /admin/tools`）返回的 `lastUsedTime` 字段（该接口由 `AdminServiceImpl#queryToolUsageStats` 提供，查询时包含 `create_time` 字段）。

---

### 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "totalCalls": 256,
    "toolUsage": [
      {
        "toolName": "readFile",
        "callCount": 120,
        "successCount": 118,
        "failCount": 2,
        "lastUsedTime": 0
      },
      {
        "toolName": "searchCode",
        "callCount": 85,
        "successCount": 80,
        "failCount": 5,
        "lastUsedTime": 0
      },
      {
        "toolName": "listDir",
        "callCount": 51,
        "successCount": 51,
        "failCount": 0,
        "lastUsedTime": 0
      }
    ]
  }
}
```

> **空数据场景**：无任何工具调用记录时，`totalCalls=0`、`toolUsage` 返回空数组 `[]`。

### 错误码说明

| code | errMsg | 含义 | 场景 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | 成功 | — |
| `401000` | 令牌不能为空 | 缺 token（HTTP 401） | 未携带 Authorization |
| `401003` | 登录状态已过期！ | Redis 登录态过期（HTTP 401） | token 过期 |
| `401004` | 令牌验证失败！ | 越权访问（HTTP 401） | C 端 token 访问 `/admin/**` |
| `404000` | 服务未找到 | chat-service 不可用（HTTP 404） | chat-service 未注册到 Nacos |
| `500000` | 服务繁忙请稍后重试 | 服务异常（HTTP 500） | Feign 调用异常或数据库异常 |

> 当 chat-service Feign 调用返回非 `200000` 时，admin-service 抛出 `ServiceException("获取工具使用统计失败")`，HTTP 200 + `code=500001`。

## 前端注意事项

1. **数据来源**：统计来自 `sys_ai_operation_log.tool_calls` 字段（JSON 数组），不是 `sys_ai_conversation` 表。仅统计 `tool_calls` 非空、非 `''`、非 `'[]'` 的记录。

2. **tool_calls 字段已修复**：新对话中的 Agent 工具调用会被 `ToolCallRecorder` 包装层捕获并写入 `sys_ai_operation_log.tool_calls` 字段，统计可正常累加。修复前的历史对话若未写入 `tool_calls`，则不计入统计。

3. **lastUsedTime 固定为 0**：本接口返回的 `lastUsedTime` 字段当前固定为 `0`（占位），不反映真实最后使用时间。如需展示工具最后使用时间，请使用「工具列表」接口（`GET /admin/tools`）返回的 `lastUsedTime` 字段。前端展示本接口数据时建议隐藏或忽略 `lastUsedTime` 字段。

4. **工具名称**：`toolName` 来自 `tool_calls` JSON 的 `name` 字段，为 Spring AI `@Tool` 注解的方法名（小驼峰，如 `readFile`、`searchCode`），不是工具类名（如 `ReadFileTool`）。

5. **排序规则**：`toolUsage` 按 `callCount` 降序排序。调用次数相同时的次级排序由 Java `Comparator` 稳定排序决定。

6. **success/fail 互斥**：`successCount + failCount = callCount`。`success` 字段为 `true` 计入成功，否则计入失败（含 `null`、`false`）。

7. **缓存说明**：本接口走 Redis 缓存，TTL=300 秒（5 分钟），缓存 key 为 `stats:tools`，无参数。

8. **与工具管理接口的区别**：
   - 本接口（统计）：从 `sys_ai_operation_log.tool_calls` 聚合，返回调用次数、成功/失败次数。
   - 工具列表接口（`GET /admin/tools`）：从 `sys_ai_operation_log` 解析 + `sys_argument` 读启用状态，返回启用状态、描述、使用次数、最后使用时间。

9. **无审计/限流注解**：本接口为只读查询，未标注 `@LogAction` 和 `@RateLimit`。

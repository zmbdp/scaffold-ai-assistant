# 操作日志

## 功能介绍

操作日志接口用于在 B 端管理后台查询 AI 调用链路日志，记录每次 AI 调用（对话、检索、向量嵌入、重排序等）的完整链路信息，用于 AI 调试、效果评估和问题排查。

**数据来源**：`sys_ai_operation_log` 表（AI 调用链路日志，由 AI 编排层手动埋点写入）。

**职责边界**：
- 本接口查询的是 **AI 调用链路日志**（`sys_ai_operation_log` 表），记录 AI 模型调用的 Prompt、响应、Token 消耗、工具调用链路等。
- B 端管理操作审计（知识源 CRUD、配置修改等）由 `@LogAction` 注解自动记录到 `operation_log` 表，**不在本接口查询范围**。

**列表页仅返回摘要信息**：为避免响应体过大，列表接口清空 `prompt`/`response`/`toolCalls` 三个大字段。完整链路详情通过「AI 调用指标与详情」接口（`GET /admin/statistics/ai-metrics/{operationId}`）查看。

**过滤参数已全部生效**：支持 `operationType`、`model`、`status`、`startDate`、`endDate` 多条件组合过滤，可同时传多个参数做交集查询。

---

## 接口详情

### 获取 AI 调用链路日志列表（分页）

| 项 | 值 |
| --- | --- |
| 路径 | `GET /admin/knowledge/logs/list` |
| 认证 | 必须携带 B 端 JWT（`userFrom=sys`），见总说明第三章 |
| Content-Type | `application/json` |

#### 路径说明

本 Controller 路径为 `/knowledge/logs`（而非 `/operation-log`），是因为 B 端管理界面将 AI 操作日志查询入口放在知识库管理模块下。底层仍通过 Feign 调用 chat-service 的 `OperationLogApi#getLogs()` 接口（chat-service 端路径为 `/operation-log/list`）。

#### 调用链路

前端请求 `/admin/knowledge/logs/list` → 网关 StripPrefix=1 → admin-service `LogController#listLogs(...)` → Feign 调用 chat-service `OperationLogApi#getLogs(...)` → `OperationLogController#getLogs(...)` → `AdminServiceImpl#listOperationLogs(...)`。

---

### 请求参数

| 参数名 | 类型 | 必传 | 默认值 | 校验规则 | 含义 |
| --- | --- | --- | --- | --- | --- |
| `pageNo` | Integer | 否 | `1` | Service 层兜底：为 `null` 或 `< 1` 时取 `1` | 页码，从 1 开始 |
| `pageSize` | Integer | 否 | `20` | Service 层兜底：为 `null` 或 `< 1` 时取 `20` | 每页大小 |
| `operationType` | String | 否 | 不传则不过滤 | 非空字符串才生效 | AI 操作类型过滤，见下方枚举 |
| `model` | String | 否 | 不传则不过滤 | 非空字符串才生效 | 模型名称过滤（精确匹配，如 `deepseek-v4-flash`） |
| `status` | String | 否 | 不传则不过滤 | 非空字符串才生效 | 调用状态过滤，见下方枚举 |
| `startDate` | Long | 否 | 不传则不过滤 | BIGINT `YYYYMMDD` | 起始日期，过滤 `create_date >= startDate` |
| `endDate` | Long | 否 | 不传则不过滤 | BIGINT `YYYYMMDD` | 结束日期，过滤 `create_date <= endDate` |

> **过滤参数生效说明**：`operationType`/`model`/`status`/`startDate`/`endDate` 均为可选条件，各自独立生效（Spring `StringUtils.hasText` 判空 / `null` 判空）。多个条件同时传时为 AND 关系（交集查询）。均不传时返回全部日志（按创建时间降序）。
>
> **model 精确匹配**：`model` 参数为精确匹配（SQL `eq`），不支持模糊查询。如需查询 `deepseek-v4-flash` 模型的日志，需传入完整模型名。

---

### 请求示例

**示例 1：查询全部日志（仅分页，不过滤）**

```
GET /admin/knowledge/logs/list?pageNo=1&pageSize=20
Authorization: Bearer {accessToken}
```

**示例 2：按操作类型 + 状态 + 日期范围组合过滤**

```
GET /admin/knowledge/logs/list?pageNo=1&pageSize=20&operationType=CHAT&status=FAILED&startDate=20260701&endDate=20260723
Authorization: Bearer {accessToken}
```

**示例 3：按模型过滤**

```
GET /admin/knowledge/logs/list?pageNo=1&pageSize=20&model=deepseek-v4-flash
Authorization: Bearer {accessToken}
```

**示例 4：按日期范围过滤**

```
GET /admin/knowledge/logs/list?pageNo=1&pageSize=20&startDate=20260701&endDate=20260723
Authorization: Bearer {accessToken}
```

### 响应结构

返回 `Result<BasePageVO<OperationLogVO>>`，`data` 为分页结果。

**BasePageVO 字段**：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `totals` | Integer | 总记录数 |
| `totalPages` | Integer | 总页数 |
| `list` | List&lt;OperationLogVO&gt; | 操作日志列表（摘要信息） |

**OperationLogVO 字段（列表页）**：

| 字段 | 类型 | 含义 | 列表页是否返回 |
| --- | --- | --- | --- |
| `id` | Long | 日志ID | ✅ 返回 |
| `userId` | Long | 调用用户ID | ✅ 返回 |
| `userFrom` | String | 用户来源（`sys`/`app`） | ✅ 返回 |
| `conversationId` | Long | 关联对话ID | ✅ 返回 |
| `operationType` | String | AI 操作类型 | ✅ 返回 |
| `model` | String | 模型名称 | ✅ 返回 |
| `prompt` | String | 完整 Prompt（含 RAG 上下文） | ❌ 列表页清空为 `null` |
| `response` | String | LLM 完整响应内容 | ❌ 列表页清空为 `null` |
| `toolCalls` | List&lt;ToolCallDetail&gt; | 工具调用链路 | ❌ 列表页清空为 `null` |
| `promptTokens` | Integer | Prompt Token 数 | ✅ 返回 |
| `completionTokens` | Integer | 响应 Token 数 | ✅ 返回 |
| `totalTokens` | Integer | 总 Token 数 | ✅ 返回 |
| `responseTime` | Integer | 响应耗时（毫秒） | ✅ 返回 |
| `status` | String | 调用状态（`SUCCESS`/`FAILED`/`TIMEOUT`） | ✅ 返回 |
| `errorMsg` | String | 失败原因 | ✅ 返回 |
| `createDate` | Long | 创建日期（`YYYYMMDD`） | ✅ 返回 |
| `createTime` | LocalDateTime | 创建时间（精确到秒，序列化为 `yyyy-MM-dd HH:mm:ss`） | ✅ 返回 |

> **列表页字段清空说明**：chat-service 端 `OperationLogController#convertToOperationLogVOPage` 在转换时显式将 `prompt`、`response`、`toolCalls` 三个字段设为 `null`，避免列表响应过大。完整链路详情通过 `GET /admin/statistics/ai-metrics/{operationId}` 查看。

### 枚举值

#### operationType（AI 操作类型）

| 值 | 含义 | 数据状态 |
| --- | --- | --- |
| `CHAT` | AI 对话 | **当前版本唯一有实际日志数据的类型**，由对话流程记录 |
| `RETRIEVE` | RAG 检索 | 枚举已声明，当前代码尚未写入日志数据 |
| `EMBEDDING` | 向量嵌入 | 枚举已声明，当前代码尚未写入日志数据 |
| `RERANK` | 重排序 | 枚举已声明，当前代码尚未写入日志数据 |

> **重要**：当前版本只有 `CHAT` 类型有实际日志数据写入（由对话流程记录）。查询时传 `CHAT` 可查到数据，传其他值结果为空。

#### status（调用状态）

| 值 | 含义 |
| --- | --- |
| `SUCCESS` | 成功 |
| `FAILED` | 失败 |
| `TIMEOUT` | 超时 |

### 错误码说明

| code | errMsg | 含义 | 场景 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | 成功 | — |
| `401000` | 令牌不能为空 | 缺 token（HTTP 401） | 未携带 Authorization |
| `401003` | 登录状态已过期！ | Redis 登录态过期（HTTP 401） | token 过期 |
| `401004` | 令牌验证失败！ | 越权访问（HTTP 401） | C 端 token 访问 `/admin/**` |
| `404000` | 服务未找到 | chat-service 不可用（HTTP 404） | chat-service 未注册到 Nacos |
| `500000` | 服务繁忙请稍后重试 | 服务异常（HTTP 500） | Feign 调用异常或数据库异常 |

> 当 chat-service Feign 调用返回非 `200000` 时，admin-service 抛出 `ServiceException`，HTTP 200 + `code=500001`。

## 前端注意事项

1. **过滤参数已全部生效**：`operationType`/`model`/`status`/`startDate`/`endDate` 五个过滤参数均已生效，支持多条件组合（AND 交集）。均不传时返回全部日志。

2. **列表与详情的职责分离**：
   - **列表页（本接口）**：返回摘要信息，`prompt`/`response`/`toolCalls` 三个大字段为 `null`。
   - **详情页**：通过 `GET /admin/statistics/ai-metrics/{operationId}` 查看，返回完整字段。前端列表点击某条记录后，取 `id` 调用详情接口获取完整链路。

3. **operationType 当前仅 CHAT 有数据**：虽然枚举声明了 `CHAT`/`RETRIEVE`/`EMBEDDING`/`RERANK` 四种类型，但当前版本只有 `CHAT` 类型有实际日志数据写入。前端操作类型下拉框可保留四个选项，但需提示用户其他类型查询结果可能为空。

4. **model 精确匹配**：`model` 参数为精确匹配，不支持模糊查询。前端可提供模型名输入框或从已有日志的 `model` 字段提取候选值。

5. **日期格式**：`startDate`/`endDate` 为 BIGINT `YYYYMMDD` 格式（如 `20260701`），过滤的是 `create_date` 字段（非 `create_time`）。`create_date` 为 Long 类型存储的日期整数。

6. **排序规则**：按 `createTime` 降序（`ORDER BY create_time DESC`），最新日志在前。

7. **分页参数兜底**：`pageNo` 为 `null` 或 `< 1` 时取 `1`；`pageSize` 为 `null` 或 `< 1` 时取 `20`。前端可不传分页参数，使用默认值。

8. **路径前缀说明**：本接口路径为 `/admin/knowledge/logs/list`（在知识库模块下），而非 `/admin/operation-log/list`。这是 B 端管理界面的入口归类决定，底层仍调用 chat-service 的 `/operation-log/list`。

9. **无审计/限流注解**：本接口为只读查询，未标注 `@LogAction`（不写入管理审计）和 `@RateLimit`（不限流）。

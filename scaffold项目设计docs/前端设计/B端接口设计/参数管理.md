# 参数管理

> 模块：参数管理 / 系统参数（`sys_argument` 表）
>
> 适用对象：B 端管理系统（`userFrom=sys` 的系统用户）
>
> 公共约定（网关地址、JWT 认证、统一响应结构 `Result<T>`、分页结构 `BasePageVO<T>`、错误码、枚举值等）见 `总说明.md`。

---

## 功能介绍

系统参数表 `sys_argument` 用于存储系统级配置项，采用 `configKey`（参数业务主键）+ `value`（参数值）结构。常见用途包括 AI 工具开关（如 `ai.tool.*.enabled`）、模型配置、各类业务开关等。其他微服务通过内部 Feign 接口按 `configKey` 读取参数值，B 端管理后台通过本模块接口维护参数。

本模块提供系统参数的**新增、分页列表查询、编辑**三个接口。

> **重要**：
> - 当前版本**没有参数删除接口**。参数新增后只能编辑 `name`/`value`/`remark`，无法删除。
> - 参数的 `configKey`（业务主键）新增后不可修改。
> - 系统参数无状态字段（无启用/停用概念）。
>
> 本模块接口均不涉及操作审计（源码未标注 `@LogAction`）。

## 一、新增参数

### 1.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `POST /admin/argument/add` |
| 认证 | 需要 token（`Authorization: Bearer {accessToken}`），且 `userFrom=sys` |
| 权限 | 所有已登录 B 端用户（未做 `identity` 区分，`super_admin` / `platform_admin` 均可） |
| Content-Type | `application/json` |
| 操作审计 | 无 |

### 1.2 请求参数

请求体为 `ArgumentAddReqDTO`：

| 字段 | 类型 | 必传 | 校验规则 | 默认值 | 含义 |
| --- | --- | --- | --- | --- | --- |
| `configKey` | String | 是 | `@NotBlank(message = "参数业务主键不能为空")`；服务端校验全局唯一 | — | 参数业务主键（如 `ai.tool.readFile.enabled`），**新增后不可修改** |
| `name` | String | 是 | `@NotBlank(message = "参数名称不能为空")`；服务端校验全局唯一 | — | 参数名称（如「读取文件工具开关」） |
| `value` | String | 是 | `@NotBlank(message = "参数值不能为空")` | — | 参数值（如 `true`、`gpt-4` 等，统一以字符串存储） |
| `remark` | String | 否 | — | — | 备注 |

> 服务端唯一性校验：`configKey` 或 `name` 任一已存在即拒绝（`configKey` OR `name` 重复）。`value` 不做唯一性校验，可重复。

### 1.3 请求示例

```http
POST /admin/argument/add
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.....
Content-Type: application/json

{
  "configKey": "ai.tool.readFile.enabled",
  "name": "读取文件工具开关",
  "value": "true",
  "remark": "控制是否启用 readFile 工具"
}
```

### 1.4 响应结构

`Result<Long>`，`data` 为新增记录的数据库自增 ID。

### 1.5 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": 20
}
```

### 1.6 错误码说明

| code | errMsg | 触发场景 |
| --- | --- | --- |
| `400000`/`500001` | 操作失败 | 参数业务主键或参数名称重复（`configKey` 或 `name` 已存在，消息为「参数业务主键或参数名称重复」）；`@NotBlank` 校验失败 |
| `401000`/`401001`/`401003`/`401004` | token 相关 | 未携带/无效/过期/越权 token（HTTP 401） |

> 注：服务端业务异常由 `ServiceException` 抛出，前端应以 `code !== 200000` 判断失败并展示 `errMsg`。

## 二、查询参数列表

### 2.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `GET /admin/argument/list` |
| 认证 | 需要 token（`Authorization: Bearer {accessToken}`），且 `userFrom=sys` |
| 权限 | 所有已登录 B 端用户 |
| Content-Type | 无请求体（参数通过 query string 传递） |
| 操作审计 | 无 |

### 2.2 请求参数

query 参数为 `ArgumentListReqDTO`（继承 `BasePageReqDTO`，含分页字段）：

| 字段 | 类型 | 必传 | 校验规则 | 默认值 | 含义 | 查询匹配方式 |
| --- | --- | --- | --- | --- | --- | --- |
| `pageNo` | Integer | 否 | — | `1` | 页码，从 1 开始 | — |
| `pageSize` | Integer | 否 | — | `10` | 每页大小 | — |
| `configKey` | String | 否 | — | `null` | 参数业务主键 | 精确匹配（`config_key =`） |
| `name` | String | 否 | — | `null` | 参数名称 | 模糊匹配（`name like concat('%', ?, '%')`） |

> 该接口使用 `@Validated` 但 DTO 字段无校验注解，参数全部可空。全部不传则分页返回全部参数。

### 2.3 请求示例

按名称模糊查询：

```http
GET /admin/argument/list?pageNo=1&pageSize=10&name=工具
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.....
```

按 configKey 精确查询：

```http
GET /admin/argument/list?pageNo=1&pageSize=10&configKey=ai.tool.readFile.enabled
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.....
```

查询全部（仅分页）：

```http
GET /admin/argument/list?pageNo=1&pageSize=20
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.....
```

### 2.4 响应结构

`Result<BasePageVO<ArgumentVO>>`，外层 `BasePageVO` 字段：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `totals` | Integer | 总记录数 |
| `totalPages` | Integer | 总页数 |
| `list` | List&lt;ArgumentVO&gt; | 数据列表 |

列表元素 `ArgumentVO` 字段：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `id` | Long | 自增主键 |
| `name` | String | 参数名称 |
| `configKey` | String | 参数业务主键 |
| `value` | String | 参数值 |
| `remark` | String | 备注 |

### 2.5 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "totals": 3,
    "totalPages": 1,
    "list": [
      {
        "id": 20,
        "name": "读取文件工具开关",
        "configKey": "ai.tool.readFile.enabled",
        "value": "true",
        "remark": "控制是否启用 readFile 工具"
      },
      {
        "id": 21,
        "name": "默认对话模型",
        "configKey": "ai.chat.defaultModel",
        "value": "gpt-4",
        "remark": "C端对话默认使用的模型"
      },
      {
        "id": 22,
        "name": "搜索代码工具开关",
        "configKey": "ai.tool.searchCode.enabled",
        "value": "false",
        "remark": "控制是否启用 searchCode 工具"
      }
    ]
  }
}
```

无匹配数据时：

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "totals": 0,
    "totalPages": 0,
    "list": []
  }
}
```

### 2.6 错误码说明

| code | errMsg | 触发场景 |
| --- | --- | --- |
| `400006` | 参数类型不匹配 | `pageNo`/`pageSize` 传非数字 |
| `401000`/`401001`/`401003`/`401004` | token 相关 | 未携带/无效/过期/越权 token（HTTP 401） |

> 该接口为纯查询，不主动抛业务错误。无匹配数据时 `totals=0`、`list=[]`。

## 三、编辑参数

### 3.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `POST /admin/argument/edit` |
| 认证 | 需要 token（`Authorization: Bearer {accessToken}`），且 `userFrom=sys` |
| 权限 | 所有已登录 B 端用户 |
| Content-Type | `application/json` |
| 操作审计 | 无 |

### 3.2 请求参数

请求体为 `ArgumentEditReqDTO`：

| 字段 | 类型 | 必传 | 校验规则 | 默认值 | 含义 |
| --- | --- | --- | --- | --- | --- |
| `configKey` | String | 是 | `@NotBlank(message = "参数业务主键不能为空")`；作为定位主键 | — | 参数业务主键（**不可修改**，仅用于定位） |
| `name` | String | 是 | `@NotBlank(message = "参数名称不能为空")`；服务端校验除自身外全局唯一 | — | 参数名称（可修改） |
| `value` | String | 是 | `@NotBlank(message = "参数值不能为空")` | — | 参数值（可修改） |
| `remark` | String | 否 | — | — | 备注（仅当传入非空值时才更新） |

> 服务端逻辑：
> - 根据 `configKey` 查询记录，不存在则抛「参数业务主键不存在」。
> - 校验 `name` 唯一性（排除当前 `configKey` 自身），若其他记录已使用该 `name` 则抛「参数名称已存在」。
> - `configKey` 不会被修改（编辑仅更新 `name`/`value`/`remark`）。
> - `name` 和 `value` 总是更新（必传字段）。
> - `remark` 仅当传入非空（`StringUtil.isNotBlank`）时才覆盖原值，否则保留原备注。

### 3.3 请求示例

```http
POST /admin/argument/edit
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.....
Content-Type: application/json

{
  "configKey": "ai.tool.readFile.enabled",
  "name": "读取文件工具开关",
  "value": "false",
  "remark": "更新：临时关闭该工具"
}
```

### 3.4 响应结构

`Result<Long>`，`data` 为编辑记录的数据库 ID。

### 3.5 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": 20
}
```

### 3.6 错误码说明

| code | errMsg | 触发场景 |
| --- | --- | --- |
| `400000`/`500001` | 操作失败 | 参数业务主键不存在（`configKey` 查不到）；参数名称已存在（`name` 与其他记录重复）；`@NotBlank` 校验失败 |
| `401000`/`401001`/`401003`/`401004` | token 相关 | 未携带/无效/过期/越权 token（HTTP 401） |

> 注：「参数业务主键不存在」「参数名称已存在」由 `ServiceException` 抛出，前端应以 `code !== 200000` 判断失败并展示 `errMsg`。

## 四、枚举值

系统参数**无固定枚举字段**。`configKey` 和 `value` 均为自由字符串，由业务约定。常见 `configKey` 命名风格示例：

| configKey 示例 | 含义 | value 示例 |
| --- | --- | --- |
| `ai.tool.readFile.enabled` | 读取文件工具开关 | `true` / `false` |
| `ai.tool.searchCode.enabled` | 搜索代码工具开关 | `true` / `false` |
| `ai.chat.defaultModel` | 默认对话模型 | `gpt-4` |

> 上述仅为命名风格示例，实际可用 `configKey` 由业务方约定并入库。`value` 统一以字符串存储，消费方按需自行解析为布尔/数字等类型。

## 五、前端注意事项

1. **无删除接口**：参数当前无删除接口，新增后只能编辑 `name`/`value`/`remark`，无法删除。前端「参数管理」列表不应提供删除按钮。
2. **configKey 不可修改**：编辑时 `configKey` 仅用于定位记录，服务端不会更新它。前端编辑表单中 `configKey` 字段应设为只读/禁用。
3. **name 唯一性**：`name`（参数名称）全局唯一，新增和编辑都会校验。编辑时排除自身后判断重复。
4. **value 无格式约束**：`value` 为字符串，`@NotBlank` 校验非空即可，无类型/格式限制。前端展示时按字符串处理；若业务需要布尔/数字，前端可做格式提示但后端不做校验。
5. **remark 空值处理**：编辑时 `remark` 仅当传入非空字符串才更新；传空串不会清空原备注。如需清空需后端调整。
6. **name/value 必传**：编辑时 `name` 和 `value` 均为必传（`@NotBlank`），总会被更新。前端编辑表单这两个字段不可留空。
7. **列表为 GET 请求**：`GET /admin/argument/list` 参数走 query string，注意 `name` 模糊查询时的 URL 编码（中文需 `encodeURIComponent`）。
8. **configKey 精确匹配**：列表查询 `configKey` 为精确匹配，不支持模糊查询。若需按 `configKey` 前缀搜索（如查所有 `ai.tool.*` 参数），当前接口不支持，需后端调整或前端拉取全量后前端过滤。
9. **参数生效方式**：参数值由其他微服务通过内部 Feign 接口按 `configKey` 实时读取，编辑后即时生效（无需重启）。前端编辑成功后可提示「已生效」。
10. **错误码判断**：服务端业务异常（如「参数业务主键或参数名称重复」「参数业务主键不存在」「参数名称已存在」）可能以 `500001` 返回，前端应统一以 `code !== 200000` 判断失败并展示 `errMsg`。

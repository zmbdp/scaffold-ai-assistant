# 工具列表与配置

> 模块：Agent 工具列表查询、工具启用状态更新
>
> 公共约定（网关地址、JWT 认证、统一响应结构 `Result<T>`、错误码、操作审计）见 `总说明.md`，本文档不再重复。

## 功能介绍

本模块用于管理 AI Agent 在对话中可调用的工具，包含两个接口：

- **获取工具列表**：返回全部已注册工具的名称、描述、启用状态、使用统计（最后使用时间、累计使用次数），不分页。
- **更新工具配置**：更新单个工具的**启用状态**，更新后立即生效（启用则重新注册到 Agent 工具池，禁用则从工具池移除）。

**工具命名规则（重要）**：工具名使用 `@Tool` 注解方法的**方法名**（小驼峰），如 `readFile`、`searchCode`，**不是**类名（如 `ReadFileTool`）。Spring AI 默认以 `@Tool` 方法名作为工具调用名称。本系统共注册 7 个工具，工具名与 `sys_argument` 表的 `ai.tool.{toolName}.enabled` 配置项一一对应。

**职责边界**：

- 本接口**仅支持启用/禁用**工具（更新 `ai.tool.{name}.enabled` 配置项）。
- 工具的运行时参数（如 `knowledge.max-file-size`、`knowledge.allowed-paths`、`knowledge.base-path`、`scaffold.tool.timeout` 等）统一通过 **Nacos 配置 + `@RefreshScope`** 管理，**不通过本接口修改**。

**调用链路**：前端 `GET/PUT /admin/tools` → 网关 StripPrefix=1 → admin-service `ToolController` → Feign `ToolsApi` → chat-service `ToolsController` → `AdminServiceImpl` → `ToolRegistryService`。

## 已注册工具清单

系统共注册 7 个工具，工具名固定（来自 `ToolRegistryService.ALL_TOOL_NAMES`）：

| 工具 name | @Tool 方法 | 工具类 | 功能 |
| --- | --- | --- | --- |
| `readFile` | `readFile(String filePath)` | `ReadFileTool` | 读取指定文件内容 |
| `searchCode` | `searchCode(String keyword, Integer limit)` | `SearchCodeTool` | 在代码库中搜索类或方法 |
| `listDir` | `listDir(String dirPath)` | `ListDirTool` | 列出指定目录结构 |
| `searchInFile` | `searchInFile(String filePath, String keyword)` | `SearchInFileTool` | 在文件中搜索关键字 |
| `nacosConfig` | `nacosConfig(String dataId, String group, String env)` | `NacosConfigTool` | 查询 Nacos 配置项内容 |
| `compareConfig` | `compareConfig(String dataId, String env1, String env2)` | `CompareConfigTool` | 对比不同环境配置差异 |
| `preDeployCheck` | `preDeployCheck(String env)` | `PreDeployCheckTool` | 部署前静态配置检查 |

> 各工具的测试入参详见 `工具测试.md`。

## 一、获取工具列表

### 1.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `GET /admin/tools` |
| 认证 | 需要 B 端 token（`userFrom=sys`） |
| 操作审计 | 无（查询接口不记录 `@LogAction`） |
| Content-Type | 无需请求体 |
| 权限 | B 端登录用户均可访问 |

### 1.2 请求参数

无（不分页，返回全部 7 个工具）。

### 1.3 请求示例

```http
GET /admin/tools HTTP/1.1
Authorization: Bearer {accessToken}
```

### 1.4 响应结构

`Result<List<ToolVO>>`，`data` 为 `ToolVO` 数组。`ToolVO` 字段：

| 字段 | 类型 | 含义 | 说明 |
| --- | --- | --- | --- |
| `name` | String | 工具名称 | `@Tool` 方法名（小驼峰），如 `readFile` |
| `description` | String | 工具描述 | 通过反射读取 `@Tool` 注解的 `description` 属性；工具未注册成功时为 `null` |
| `enabled` | Boolean | 是否启用 | 来自 `ai.tool.{name}.enabled` 配置项。配置项缺失时**默认启用**（与 `sys_argument` 预置数据一致） |
| `lastUsedTime` | Long | 最后使用时间 | 毫秒级时间戳；从未使用过为 `null` |
| `usedCount` | Integer | 累计使用次数 | 从 `sys_ai_operation_log.tool_calls` 字段解析统计；从未使用过为 `0` |

> **使用统计来源**：从 `sys_ai_operation_log` 表中 `tool_calls` 非空的记录解析 JSON（格式 `[{"name":"readFile","success":true,"duration":45,"summary":"..."}]`），按 `name` 分组统计次数和最后使用时间。统计依赖历史对话日志，工具未在对话中被调用过时 `usedCount=0`、`lastUsedTime=null`。

### 1.6 错误码说明

| code | errMsg | 触发场景 |
| --- | --- | --- |
| `200000` | 操作成功 | 正常返回 |
| `500000` | 获取工具列表失败 / 服务繁忙请稍后重试 | Feign 调用 chat-service 失败，或 chat-service 内部异常 |

> 本接口不触发 AI 模块专属错误码。无工具注册时返回空数组 `[]`（不会发生，系统预置 7 个工具）。

### 1.7 枚举值

工具名枚举见上文「已注册工具清单」（共 7 个合法值）。`enabled` 为布尔值。

### 1.8 前端注意事项

- 列表固定返回 7 个工具，**不分页**，前端可直接全量渲染。
- `description` 来源于工具类 `@Tool` 注解，可直接作为工具说明展示给用户。
- `enabled` 缺省策略为「默认启用」：当 `sys_argument` 中对应配置项不存在或 Feign 查询失败时，工具按启用处理。因此即使配置缺失，列表也会显示 `enabled=true`。
- `usedCount` / `lastUsedTime` 依赖对话日志，新部署或无对话历史时分别为 `0` / `null`。

## 二、更新工具配置

### 2.1 接口信息

| 项目 | 内容 |
| --- | --- |
| 路径 | `PUT /admin/tools/{name}` |
| 认证 | 需要 B 端 token（`userFrom=sys`） |
| 操作审计 | **有**：`@LogAction(value = "更新工具配置", module = "tools", recordParams = true)`，记录到 `operation_log` 表 |
| Content-Type | `application/json` |
| 权限 | B 端登录用户均可访问 |

### 2.2 请求参数

#### 路径参数

| 参数名 | 类型 | 必传 | 含义 |
| --- | --- | --- | --- |
| `name` | String | 是 | 工具名称（`@Tool` 方法名，小驼峰），如 `readFile`。合法值见上文「已注册工具清单」 |

#### 请求体（`ToolConfigDTO`）

| 字段名 | 类型 | 必传 | 校验规则（注解原文） | 默认值 | 含义 |
| --- | --- | --- | --- | --- | --- |
| `enabled` | Boolean | 否 | 无（DTO 上无校验注解） | 无 | 是否启用工具。持久化到 `ai.tool.{name}.enabled` |

> **请求体仅含 `enabled` 一个字段**。`ToolConfigDTO` 不包含工具运行时参数（如 `maxFileSize`、`basePath`、`pathWhitelist` 等），这些参数统一在 Nacos 管理。请求体若传其他字段会被忽略。

### 2.3 请求示例

禁用 `compareConfig` 工具：

```http
PUT /admin/tools/compareConfig HTTP/1.1
Authorization: Bearer {accessToken}
Content-Type: application/json

{
  "enabled": false
}
```

启用 `readFile` 工具：

```http
PUT /admin/tools/readFile HTTP/1.1
Authorization: Bearer {accessToken}
Content-Type: application/json

{
  "enabled": true
}
```

### 2.4 响应结构

`Result<Void>`，无 `data`。

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `code` | int | 响应码，成功为 `200000` |
| `errMsg` | String | 消息描述 |
| `data` | null | 无业务数据 |

### 2.5 响应示例

成功：

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": null
}
```

### 2.6 错误码说明

| code | errMsg | 触发场景 | HTTP 状态码 |
| --- | --- | --- | --- |
| `200000` | 操作成功 | 更新成功 |
| `400000` | 工具名称不能为空 | `name` 路径参数为空（Service 层 `StringUtils.hasText` 校验） | 400 |
| `500000` | 更新工具配置失败 / 服务繁忙请稍后重试 | Feign 调用 chat-service 失败，或 chat-service 内部异常 | 500 |

> **关于不存在的工具名**：服务端**不会**校验 `name` 是否在 7 个合法工具名中。若传入不存在的工具名（如 `readFileTool` 类名形式，或任意字符串），后端会尝试更新 `ai.tool.{name}.enabled` 配置项：若 `sys_argument` 中无此 `configKey`，则跳过更新并打印警告日志，同时刷新一个不存在的工具缓存（无副作用），最终仍返回 `200000`。**因此前端必须从「已注册工具清单」中选择合法工具名**，避免传入非法值导致「看似成功但实际未生效」。

### 2.7 枚举值

`name` 合法值（共 7 个）：`readFile` / `searchCode` / `listDir` / `searchInFile` / `nacosConfig` / `compareConfig` / `preDeployCheck`。

`enabled` 为布尔值（`true` / `false`）。

### 2.8 前端注意事项

- **工具名大小写敏感**，必须使用小驼峰方法名（`readFile`），**不要**使用类名（`ReadFileTool`）。传入类名形式不会报错但不会生效。
- 更新**立即生效**：`enabled` 从 `false` 改为 `true` 时，工具被 `ToolRegistryService` 重新注册到 Agent 工具池，后续对话即可调用；反之则从工具池移除。无需重启服务。
- 本接口仅切换启用状态，**不能**修改工具的运行时参数（如文件大小限制、路径白名单等）。如需调整这些参数，请在 Nacos 的 `knowledge.*` / `scaffold.tool.*` 配置项中修改。
- 建议前端在工具列表页提供开关组件（Switch），切换时调用本接口，成功后刷新列表的 `enabled` 状态。
- 前端应限制 `name` 只能从 7 个合法值中选择（下拉/列表点击），避免用户手动输入非法工具名。

# C 端接口设计 - 对话模块

> 公共约定（网关地址、认证机制、响应结构、错误码、日期格式等）见 [./总说明.md](./总说明.md)。

## 功能介绍

对话模块提供 C 端 AI 流式对话能力，对应后端 `ChatPortalController`（前缀 `/chat`，前端经网关访问 `/portal/chat/**`）。

主要使用场景：
- **聊天主界面**：用户输入问题，AI 通过 SSE 流式逐字输出回答
- **多轮对话**：首次对话不传 `sessionId`，从结束帧获取后端生成的 `sessionId`，后续对话回传同一 `sessionId` 串联上下文
- **图文提问**：用户上传图片（先通过文件上传获取 URL）+ 文字提问，调用视觉模型回答
- **RAG 知识增强**：对话前自动检索知识库相关文档拼入 Prompt，提供项目上下文（检索失败会降级为空上下文继续对话，不中断流）

> 后端流程：`ChatPortalController` 接收请求 → `ChatPortalServiceImpl` 执行业务编排（解析 JWT 提取用户 → 生成/校验 sessionId → Feign 调用 RAG 检索 → 拼接 Prompt → WebClient 调用 chat-service SSE 端点 → 透传 `Flux<String>` 给前端）。

## 接口列表

| # | 路径 | 方法 | 认证 | 特殊说明 |
| --- | --- | --- | --- | --- |
| 1 | `/portal/chat/completions/stream` | POST | 是 | `@RateLimit`，SSE 流 |
| 2 | `/portal/chat/image/completions/stream` | POST | 是 | `@RateLimit`，SSE 流 |

> **注意**：两个接口返回 `text/event-stream`（SSE 流），**不是** `Result<T>` 包装结构。每帧为 JSON 字符串，需逐帧 `JSON.parse`。

## SSE 帧格式总览

两个对话接口的 SSE 帧格式完全一致，共三种帧类型。帧定义来自 chat-service 的 `ChatServiceImpl`（portal-service 透传 chat-service 的帧，仅 WebClient 调用失败/超时时由 portal-service 自行生成错误帧）。

### ① 内容帧（流式输出 AI 回答片段）

```json
{"chunk": "这是", "done": false}
{"chunk": "AI", "done": false}
{"chunk": "的回答", "done": false}
```

| 字段 | Java 类型 | 前端类型 | 含义 |
| --- | --- | --- | --- |
| `chunk` | String | string | 本次输出的文本片段（可能为空字符串 `""`） |
| `done` | Boolean | boolean | 固定 `false` |

### ② 结束帧（流结束）

```json
{"chunk": "", "done": true, "sessionId": "1894736281934592", "model": "deepseek-v4-flash"}
```

| 字段 | Java 类型 | 前端类型 | 含义 |
| --- | --- | --- | --- |
| `chunk` | String | string | 固定空字符串 `""` |
| `done` | Boolean | boolean | 固定 `true` |
| `sessionId` | String | string | 会话 ID（**首次对话时前端应保存此值用于后续多轮对话**） |
| `model` | String | string | 实际使用的模型名称 |

### ③ 错误帧（AI 服务异常，流终止）

```json
{"error": "AI 服务连接失败", "code": 500023, "done": true}
{"error": "AI 服务响应超时", "code": 500024, "done": true}
```

| 字段 | Java 类型 | 前端类型 | 含义 |
| --- | --- | --- | --- |
| `error` | String | string | 错误信息 |
| `code` | int | number | 错误码（`500023` 连接失败 / `500024` 超时） |
| `done` | Boolean | boolean | 固定 `true` |

### 帧类型判断逻辑

收到一帧后 `JSON.parse`，按字段判断：

| 判断条件 | 帧类型 | 处理方式 |
| --- | --- | --- |
| `done === false` | 内容帧 | 取 `chunk` 拼接显示 |
| `done === true` 且无 `error` 字段 | 结束帧 | 取 `sessionId` / `model`，关闭流 |
| `done === true` 且有 `error` 字段 | 错误帧 | 取 `error` / `code`，提示用户，关闭流 |

## 1. 流式文本对话

**接口**：`POST /portal/chat/completions/stream`

**认证**：需要

**限流**：`@RateLimit`（无显式参数），使用 Nacos 全局配置 `ratelimit.default-limit`（默认 60）/ `ratelimit.default-window-sec`（默认 60 秒），维度默认 `IP`。触发返回 `400020`。

**Content-Type**：`application/json`

**响应类型**：`text/event-stream`（SSE 流）

### 请求体

| 字段 | Java 类型 | 前端类型 | 必传 | 校验规则（注解原文） | 默认值 | 含义 |
| --- | --- | --- | --- | --- | --- | --- |
| `message` | String | string | 是 | `@NotBlank(message = "消息不能为空")` | — | 用户提问内容 |
| `sessionId` | String | string | 否 | 无 | 后端生成（UUID 去横线） | 会话 ID。传非空值会校验归属当前用户，不属于返回 `500032` |
| `model` | String | string | 否 | 无 | Nacos 配置的默认文本模型 | 指定模型 |
| `temperature` | Double | number | 否 | 无（约定 0-2） | `0.7`（chat-service 应用） | 温度参数 |
| `topK` | Integer | number | 否 | 无（约定 1-20） | `5`（portal-service 应用） | RAG 检索数量 |

> 请求体 DTO 为 `ChatReqDTO`（来自 `zmbdp-chat-api`）。`temperature` / `topK` 的取值范围与默认值仅为代码注释约定，未通过校验注解强制；`topK` 为空时 portal-service 使用默认值 `5`，`temperature` 透传由 chat-service 处理默认值。

### 请求示例

```json
{
  "message": "如何实现二级缓存？",
  "sessionId": "1894736281934592",
  "model": "deepseek-v4-flash",
  "temperature": 0.7,
  "topK": 5
}
```

### 响应结构

SSE 流，帧格式见 [SSE 帧格式总览](#sse-帧格式总览)。

### 业务说明

- **sessionId 处理**：
  - 不传或为空：后端生成新 `sessionId`（`UUID.randomUUID()` 去横线），通过结束帧返回。
  - 传非空值：后端通过 Feign 调用 `HistoryApi.checkSessionOwnership` 校验归属当前用户；不属于返回 `500032 会话不属于当前用户`。**Feign 调用失败时降级为校验通过**（不中断对话）。
- **RAG 检索**：portal-service 通过 Feign 调用 `ChatApi.retrieveContext` 获取相关文档，按 score 降序拼入 Prompt，限制总长度不超过 `scaffold.portal.context-max-length`（默认 8000 字符）。检索失败时使用空上下文继续对话（不中断流）。
- **对话历史**：通过 Feign 调用 `HistoryApi.getSessionHistory` 获取最近 N 轮历史拼入 Prompt（chat-service 内部按 `scaffold.rag.memory-rounds` 截取）。获取失败时使用空历史继续。
- **超时**：WebClient 调用 chat-service 超时时间默认 60 秒（`scaffold.portal.webclient-timeout`），超时返回错误帧 `500024`。
- **WebClient 调用失败**：返回错误帧 `500023 AI 服务连接失败`。

### 错误码

| code | errMsg | 场景 | 返回形式 |
| --- | --- | --- | --- |
| `401000` / `401001` / `401003` | 见总说明 | token 缺失 / 无效 / 登录态过期（网关校验） | `Result` JSON |
| `400000` | 无效的参数 | `message` 为空（触发 `@NotBlank`）/ Service 层解析 JWT 失败（`用户令牌有误`） | `Result` JSON |
| `400020` | 请求过于频繁，请稍后重试 | 触发限流 | `Result` JSON |
| `500032` | 会话不属于当前用户 | 传入的 `sessionId` 不属于当前用户 | `Result` JSON |
| `500023` | AI 服务连接失败 | WebClient 调用 chat-service 失败 | SSE 错误帧 |
| `500024` | AI 服务响应超时 | WebClient 调用超时（60 秒） | SSE 错误帧 |

> **重要**：限流（`400020`）、参数校验失败（`400000`）、sessionId 归属校验失败（`500032`）、token 失效等发生在「流返回前」的错误，由全局异常处理器返回 `Result` JSON（非 SSE 帧）；WebClient 调用 chat-service 过程中的错误（`500023` / `500024`）通过 SSE 错误帧返回。前端需根据响应 Content-Type 区分：非 `text/event-stream` 时按 `Result` JSON 解析。

## 2. 流式图文对话

**接口**：`POST /portal/chat/image/completions/stream`

**认证**：需要

**限流**：`@RateLimit`（无显式参数），同文本对话（默认 60 次/60 秒，IP 维度）。

**Content-Type**：`application/json`

**响应类型**：`text/event-stream`（SSE 流）

### 请求体

| 字段 | Java 类型 | 前端类型 | 必传 | 校验规则（注解原文） | 默认值 | 含义 |
| --- | --- | --- | --- | --- | --- | --- |
| `message` | String | string | 是 | `@NotBlank(message = "消息不能为空")` | — | 用户提问内容 |
| `images` | List&lt;String&gt; | string[] | 是 | `@NotEmpty(message = "图片不能为空")` | — | 图片 URL 列表（先通过文件上传接口获取 URL） |
| `sessionId` | String | string | 否 | 无 | 后端生成 | 会话 ID（同文本对话归属校验） |
| `model` | String | string | 否 | 无 | Nacos 配置的默认视觉模型 | 指定视觉模型 |
| `temperature` | Double | number | 否 | 无（约定 0-2） | `0.7` | 温度参数 |
| `topK` | Integer | number | 否 | 无（约定 1-20） | `5` | RAG 检索数量 |

> 请求体 DTO 为 `ChatWithImageReqDTO`（来自 `zmbdp-chat-api`）。

### 请求示例

```json
{
  "message": "这张图片里是什么？请描述一下",
  "images": [
    "https://your-bucket.oss-cn-beijing.aliyuncs.com/upload/abc123.png"
  ],
  "sessionId": "1894736281934592",
  "model": "qwen-vl-max",
  "temperature": 0.7,
  "topK": 5
}
```

### 响应结构

SSE 流，帧格式同文本对话（见 [SSE 帧格式总览](#sse-帧格式总览)）。

### 业务说明

- **图片 URL 清洗**：portal-service 入口处会清洗图片 URL，循环去除首尾的反引号、单引号、双引号等 Markdown 残留字符，并过滤掉非 `http://` / `https://` 协议的非法 URL（清洗后为空或非 http/https 的 URL 会被跳过）。
- 图文对话同样走 RAG 检索，提供项目上下文。
- 数据库仅存图片 URL 列表（JSON 数组），后端不接收也不存储图片二进制内容。
- 图片需先通过文件上传接口（见 [./文件模块.md](./文件模块.md)）获取 URL。
- sessionId 处理、超时、WebClient 失败逻辑同文本对话。

### 错误码

| code | errMsg | 场景 | 返回形式 |
| --- | --- | --- | --- |
| `401000` / `401001` / `401003` | 见总说明 | token 缺失 / 无效 / 登录态过期 | `Result` JSON |
| `400000` | 无效的参数 | `message` 为空 / `images` 为空（触发 `@NotBlank` / `@NotEmpty`）/ JWT 解析失败 | `Result` JSON |
| `400020` | 请求过于频繁，请稍后重试 | 触发限流 | `Result` JSON |
| `500032` | 会话不属于当前用户 | 传入的 `sessionId` 不属于当前用户 | `Result` JSON |
| `500023` | AI 服务连接失败 | WebClient 调用 chat-service 图文端点失败 | SSE 错误帧 |
| `500024` | AI 服务响应超时 | WebClient 调用超时（60 秒） | SSE 错误帧 |

## 前端注意事项

### 1. SSE 连接处理

- 使用 `EventSource` 或 `fetch` + `ReadableStream` 处理 SSE 流。
- 每帧是 JSON 字符串，需 `JSON.parse` 后按 `done` 字段判断帧类型（见 [帧类型判断逻辑](#帧类型判断逻辑)）。
- **收到结束帧或错误帧后关闭流连接**。

### 2. 首次对话与多轮对话

- **首次对话**：不传 `sessionId`，从结束帧获取后端生成的 `sessionId` 并保存。
- **多轮对话**：回传同一 `sessionId`，后端会校验归属当前用户并自动拼接历史上下文。
- 传入他人的 `sessionId` 会返回 `500032`。

### 3. 错误响应区分

- 流返回前的错误（token 失效、参数校验、限流、sessionId 归属校验）返回 `Result` JSON，HTTP 响应 Content-Type **不是** `text/event-stream`。
- 流过程中的错误（AI 服务连接失败、超时）通过 SSE 错误帧返回。
- 前端应根据响应 Content-Type 区分：若不是 `text/event-stream`，按 `Result` JSON 解析 `code` / `errMsg`。

### 4. 限流

- 对话接口限流阈值由 Nacos 全局配置控制（默认 60 次/60 秒，IP 维度），超限返回 `400020`。
- 限流触发时返回 `Result` JSON（非 SSE 帧）。

### 5. 图片上传

- 图文对话前，先调用 `/file/upload` 上传图片获取 URL，再将 URL 放入 `images` 数组。
- 图片 URL 必须是 `http://` 或 `https://` 协议，否则会被后端过滤掉。
- 单文件大小上限受 Spring multipart 配置控制（非代码硬编码），如需上传大文件请使用 `/file/sign` 直传。

### 6. 超时

- AI 服务超时时间默认 60 秒，超时后返回错误帧 `500024`。
- RAG 检索失败、对话历史获取失败、sessionId 归属校验 Feign 调用失败均会**降级处理**（使用空上下文 / 空历史 / 放行校验），不中断对话流。

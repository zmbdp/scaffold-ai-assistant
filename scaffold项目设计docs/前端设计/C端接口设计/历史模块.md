# C 端接口设计 - 历史模块

> 公共约定（网关地址、认证机制、响应结构、错误码、日期格式等）见 [./总说明.md](./总说明.md)。

## 功能介绍

历史模块提供 C 端对话历史的查询与删除能力，对应后端 `HistoryPortalController`（前缀 `/history`，前端经网关访问 `/portal/history/**`）。

主要使用场景：
- **历史会话列表**：分页查询当前用户的全部对话会话，每个会话聚合为一条记录（最后一条消息摘要 + 消息数量 + 使用的模型）
- **会话详情回看**：进入某个会话后，查看该会话下全部消息（用户提问 + AI 回答），按时间正序排列，含图文对话的图片 URL、RAG 引用来源
- **删除会话**：清空某条会话，同时删除 Redis 对话记忆缓存 + MySQL 记录软删除

> 后端流程：`HistoryPortalController` 接收请求 → `HistoryPortalServiceImpl` 从 JWT 解析 `userId` → Feign 调用 chat-service 的 `HistoryApi` → chat-service 的 `HistoryServiceImpl` 按 `userId` 过滤数据返回。
>
> **数据隔离**：所有历史数据均按当前登录用户的 `userId` 过滤，C 端用户只能查看 / 删除自己的会话。

## 1. 获取对话历史列表（分页）

**接口**：`GET /portal/history`

**认证**：需要

**限流 / 幂等**：无

**Content-Type**：无（GET 请求）

### 查询参数

| 参数 | Java 类型 | 前端类型 | 必传 | 校验规则 | 默认值 | 含义 |
| --- | --- | --- | --- | --- | --- | --- |
| `pageNo` | Integer | number | 否 | 无（Service 层兜底 `pageNo < 1` 时取 1） | `1` | 页码，从 1 开始 |
| `pageSize` | Integer | number | 否 | 无（Service 层兜底 `pageSize < 1` 时取 20） | `20` | 每页数量 |

> Controller 使用 `@RequestParam(value = "pageNo", defaultValue = "1")` / `@RequestParam(value = "pageSize", defaultValue = "20")`，前端不传时使用默认值。`userId` 由后端从 JWT 提取，前端无需传递。

### 请求示例

```
GET /portal/history?pageNo=1&pageSize=20
Authorization: Bearer {accessToken}
```

### 响应结构 `Result<BasePageVO<HistoryVO>>`

`data` 为 `BasePageVO<HistoryVO>`，结构见总说明 [五、分页响应结构](./总说明.md#五分页响应结构-basepagevot)。`list` 元素 `HistoryVO` 字段：

| 字段 | Java 类型 | 前端类型 | 含义 |
| --- | --- | --- | --- |
| `sessionId` | String | string | 会话 ID |
| `lastMessage` | String | string | 最后一条消息（用户提问摘要） |
| `timestamp` | Long | number | 最后消息时间（毫秒时间戳） |
| `model` | String | string | 使用的模型名称 |
| `messageCount` | Integer | number | 该会话的消息数量 |

### 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "totals": 2,
    "totalPages": 1,
    "list": [
      {
        "sessionId": "1894736281934592",
        "lastMessage": "如何实现二级缓存？",
        "timestamp": 1784900000000,
        "model": "deepseek-v4-flash",
        "messageCount": 6
      },
      {
        "sessionId": "1894736281934593",
        "lastMessage": "这张图片里是什么？",
        "timestamp": 1784900100000,
        "model": "qwen-vl-max",
        "messageCount": 4
      }
    ]
  }
}
```

> 列表按最后消息时间倒序返回。当前用户无历史时返回 `totals: 0`、`totalPages: 0`、`list: []`。

### 业务说明

- **按 sessionId 聚合**：每个会话聚合为一条 `HistoryVO`，`lastMessage` 取该会话最后一条用户提问，`messageCount` 为该会话的对话记录数。
- **数据隔离**：chat-service 按 `userId`（portal-service 从 JWT 解析）过滤，仅返回当前用户的历史。
- **分页计算**：`totalPages` 由 `BasePageDTO.calculateTotalPages(totals, pageSize)` 计算。请求页码超出总页数时返回空列表（非错误）。

### 错误码

| code | errMsg | 场景 |
| --- | --- | --- |
| `401000` / `401001` / `401003` | 见总说明 | token 缺失 / 无效 / 登录态过期（网关校验） |
| `400000` | 无效的参数 | Service 层解析 JWT 失败（`用户令牌有误`） |
| `500001` | 操作失败 | Feign 调用 chat-service 失败或返回非成功（`查询对话历史列表失败`） |

## 2. 获取会话详情

**接口**：`GET /portal/history/{sessionId}`

**认证**：需要

**限流 / 幂等**：无

**Content-Type**：无（GET 请求）

### 路径参数

| 参数 | Java 类型 | 前端类型 | 必传 | 含义 |
| --- | --- | --- | --- | --- |
| `sessionId` | String | string | 是 | 会话 ID（来自列表接口或对话结束帧） |

### 请求示例

```
GET /portal/history/1894736281934592
Authorization: Bearer {accessToken}
```

### 响应结构 `Result<HistoryDetailVO>`

| 字段 | Java 类型 | 前端类型 | 含义 |
| --- | --- | --- | --- |
| `sessionId` | String | string | 会话 ID |
| `messages` | List&lt;Message&gt; | array | 消息列表（按时间正序） |

**`Message` 子对象字段**：

| 字段 | Java 类型 | 前端类型 | 含义 |
| --- | --- | --- | --- |
| `role` | String | string | 角色：`user`（用户提问）/ `assistant`（AI 回答） |
| `content` | String | string | 消息内容 |
| `timestamp` | Long | number | 时间戳（毫秒） |
| `images` | List&lt;String&gt; | string[] | 图片 URL 列表（仅 `user` 消息，图文对话时由后端透传；纯文本对话或 `assistant` 消息为 `null`） |
| `sources` | List&lt;String&gt; | string[] | 引用来源（仅 `assistant` 消息，RAG 检索命中的文档列表；无 RAG 命中时为 `null`） |
| `model` | String | string | 模型名称（仅 `assistant` 消息） |

### 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": {
    "sessionId": "1894736281934592",
    "messages": [
      {
        "role": "user",
        "content": "如何实现二级缓存？",
        "timestamp": 1784900000000,
        "images": null,
        "sources": null,
        "model": null
      },
      {
        "role": "assistant",
        "content": "二级缓存通常使用 Caffeine + Redis 实现……",
        "timestamp": 1784900000000,
        "images": null,
        "sources": ["设计文档-缓存方案.md", "CacheConfig.java"],
        "model": "deepseek-v4-flash"
      },
      {
        "role": "user",
        "content": "这张图片里是什么？",
        "timestamp": 1784900100000,
        "images": ["https://your-bucket.oss-cn-beijing.aliyuncs.com/upload/abc123.png"],
        "sources": null,
        "model": null
      },
      {
        "role": "assistant",
        "content": "这张图片显示的是一个……",
        "timestamp": 1784900100000,
        "images": null,
        "sources": null,
        "model": "qwen-vl-max"
      }
    ]
  }
}
```

### 业务说明

- **归属校验**：chat-service 先校验 `sessionId` 归属当前 `userId`，不属于返回 `500032`。
  - **新会话判定**：`sessionId` 在数据库中不存在（新会话）时，归属校验返回 `true`（允许访问），此时返回 `messages: []`。
- **数据源**：从 MySQL `sys_ai_conversation` 表查询（非 Redis），因为表字段齐全（含 `images` / `sources` / `model`），Redis 中的 Spring AI Message 对象仅含文本，不携带业务字段。
- **消息拆分**：每条对话记录拆分为 `user`（提问）+ `assistant`（回答）两条消息；当 `answer` 为空（对话失败且无响应）时跳过 `assistant` 消息。
- **images / sources**：数据库以 JSON 数组字符串存储（如 `["url1","url2"]`），后端反序列化为 List 返回；为空或解析失败时返回 `null`。
- **时间戳**：`timestamp` 来自数据库 `create_time`（`LocalDateTime`）转换为毫秒时间戳（系统时区）。

### 错误码

| code | errMsg | 场景 |
| --- | --- | --- |
| `401000` / `401001` / `401003` | 见总说明 | token 缺失 / 无效 / 登录态过期（网关校验） |
| `400000` | 无效的参数 | Service 层解析 JWT 失败（`用户令牌有误`） |
| `500032` | 会话不属于当前用户 | `sessionId` 归属校验失败（属于其他用户） |
| `500001` | 操作失败 | Feign 调用 chat-service 失败或返回非成功（`查询会话详情失败`） |

## 3. 删除会话

**接口**：`DELETE /portal/history/{sessionId}`

**认证**：需要

**限流 / 幂等**：无

**Content-Type**：无

### 路径参数

| 参数 | Java 类型 | 前端类型 | 必传 | 含义 |
| --- | --- | --- | --- | --- |
| `sessionId` | String | string | 是 | 要删除的会话 ID |

### 请求示例

```
DELETE /portal/history/1894736281934592
Authorization: Bearer {accessToken}
```

### 响应结构 `Result<Void>`

`data` 为 `null`。

### 响应示例

```json
{
  "code": 200000,
  "errMsg": "操作成功",
  "data": null
}
```

### 业务说明

- **归属校验**：先校验 `sessionId` 归属当前 `userId`，不属于返回 `500032`。
- **双重删除**：
  1. **Redis 缓存**：删除该会话的对话记忆缓存（`chatMemoryService.clearHistory`，内部通过 MQ 广播失效多机 L1 Caffeine 缓存）。
  2. **MySQL 软删除**：将 `sys_ai_conversation` 表中该 `sessionId` 的所有记录置 `is_deleted=1`（**非物理删除**，数据仍保留在数据库，仅查询时过滤）。
- 删除后该会话不再出现在历史列表，也无法再通过详情接口查询（归属校验对软删除后的会话：数据库查不到归属用户 → 视为新会话 → 返回 `true`，但消息列表为空）。

### 错误码

| code | errMsg | 场景 |
| --- | --- | --- |
| `401000` / `401001` / `401003` | 见总说明 | token 缺失 / 无效 / 登录态过期（网关校验） |
| `400000` | 无效的参数 | Service 层解析 JWT 失败（`用户令牌有误`） |
| `500032` | 会话不属于当前用户 | `sessionId` 归属校验失败（属于其他用户） |
| `500001` | 操作失败 | Feign 调用 chat-service 失败或返回非成功（`删除会话失败`） |

## 前端注意事项

### 1. 会话列表排序与空态

- 列表按最后消息时间倒序排列（最近对话在前）。
- 无历史记录时返回 `totals: 0`、`totalPages: 0`、`list: []`，前端应展示空状态。

### 2. sessionId 来源

- 历史列表的 `sessionId` 字段。
- 对话结束帧（SSE `done: true`）返回的 `sessionId` 字段。
- 进入会话详情或删除会话时使用同一 `sessionId`。

### 3. 归属校验

- 访问 / 删除他人的 `sessionId` 会返回 `500032 会话不属于当前用户`，前端应提示错误并返回列表页。
- 访问一个数据库中不存在的 `sessionId`（新会话）不会报错，返回 `messages: []`。

### 4. 消息角色与字段差异

- `user` 消息：含 `content`（提问）、`images`（图文对话时）；`sources` / `model` 为 `null`。
- `assistant` 消息：含 `content`（回答）、`sources`（RAG 引用，可为 `null`）、`model`；`images` 为 `null`。
- 渲染消息时按 `role` 区分样式，`sources` 非空时展示「引用来源」列表。

### 5. 删除是软删除

- 删除会话为软删除（`is_deleted=1`），数据仍保留在数据库，但前端无法再查询到。
- 删除会同时清除 Redis 对话记忆缓存，后续即使重新使用同一 `sessionId` 也不会带上历史上下文。

### 6. 分页参数兜底

- `pageNo` / `pageSize` 非法（`< 1` 或不传）时后端兜底为 `1` / `20`，不会报错。
- 超出总页数的请求返回空列表（非错误）。
